Tipus d'Emmagatzematge de Dades

Introducció

En el món de l'arquitectura de dades, és fonamental entendre els diferents tipus d'emmagatzematge de dades disponibles. Cada tipus té les seves pròpies característiques, avantatges i desavantatges, i és adequat per a diferents tipus d'aplicacions i necessitats. En aquesta secció, explorarem els principals tipus d'emmagatzematge de dades, les seves característiques i exemples d'ús.

Tipus d'Emmagatzematge de Dades

Emmagatzematge en Bases de Dades Relacionals (RDBMS)

Les bases de dades relacionals són un dels tipus més comuns d'emmagatzematge de dades. Utilitzen un model basat en taules per organitzar les dades i permeten realitzar consultes complexes mitjançant SQL (Structured Query Language).

  • Característiques:

    • Estructura basada en taules.
    • Relacions entre taules mitjançant claus primàries i foranes.
    • Suport per a transaccions ACID (Atomicitat, Consistència, Aïllament, Durabilitat).
    • Escalabilitat vertical.

  • Exemples de RDBMS:

    • MySQL
    • PostgreSQL
    • Oracle Database
    • Microsoft SQL Server

  • Exemple de codi SQL:

    CREATE TABLE empleats (
    id INT PRIMARY KEY,
    nom VARCHAR(100),
    lloc VARCHAR(50),
    salari DECIMAL(10, 2)
);

INSERT INTO empleats (id, nom, lloc, salari) VALUES (1, 'Joan Pérez', 'Desenvolupador', 50000.00);

SELECT * FROM empleats WHERE lloc = 'Desenvolupador';

Emmagatzematge en Bases de Dades NoSQL

Les bases de dades NoSQL estan dissenyades per manejar grans volums de dades no estructurades o semi-estructurades. Són altament escalables i s'utilitzen en aplicacions que requereixen alta disponibilitat i rendiment.

  • Característiques:

    • No utilitzen un esquema fix.
    • Escalabilitat horitzontal.
    • Alta disponibilitat i particionament de dades.
    • Diversos models de dades (documents, clau-valor, columnes, grafs).

  • Exemples de Bases de Dades NoSQL:

    • MongoDB (documents)
    • Redis (clau-valor)
    • Cassandra (columnes)
    • Neo4j (grafs)

  • Exemple de codi MongoDB:

    db.empleats.insertOne({
    id: 1,
    nom: 'Joan Pérez',
    lloc: 'Desenvolupador',
    salari: 50000.00
});

db.empleats.find({ lloc: 'Desenvolupador' });

Emmagatzematge en Data Warehouses

Els magatzems de dades (Data Warehouses) estan dissenyats per a l'anàlisi i la generació d'informes a partir de grans volums de dades històriques. Són utilitzats principalment en intel·ligència de negocis (BI).

  • Característiques:

    • Optimitzat per a consultes analítiques.
    • Integració de dades de múltiples fonts.
    • Suport per a processament en paral·lel.
    • Models de dades dimensionals (estrelles, flocs de neu).

  • Exemples de Data Warehouses:

    • Amazon Redshift
    • Google BigQuery
    • Snowflake
    • Microsoft Azure Synapse Analytics

  • Exemple de codi SQL en un Data Warehouse:

    SELECT producte, SUM(vendes) AS total_vendes
FROM vendes
WHERE data BETWEEN '2023-01-01' AND '2023-12-31'
GROUP BY producte
ORDER BY total_vendes DESC;

Emmagatzematge en Data Lakes

Els llacs de dades (Data Lakes) permeten emmagatzemar grans volums de dades en el seu format natiu, ja siguin estructurades, semi-estructurades o no estructurades. Són ideals per a l'emmagatzematge de dades en brut que poden ser processades i analitzades posteriorment.

  • Característiques:

    • Emmagatzematge de dades en el seu format original.
    • Alta capacitat d'emmagatzematge.
    • Flexibilitat per a diferents tipus de dades.
    • Integració amb eines de processament i anàlisi de dades.

  • Exemples de Data Lakes:

    • Amazon S3
    • Azure Data Lake Storage
    • Google Cloud Storage

  • Exemple d'ús d'Amazon S3:

    aws s3 cp dades.csv s3://mi-data-lake/dades/

Conclusió

Comprendre els diferents tipus d'emmagatzematge de dades és crucial per dissenyar arquitectures de dades eficients i efectives. Cada tipus té els seus propis avantatges i desavantatges, i l'elecció del tipus adequat depèn de les necessitats específiques de l'aplicació i els requisits de rendiment. En dominar aquests conceptes, els professionals poden prendre decisions informades i optimitzar la gestió i anàlisi de dades a les seves organitzacions.
