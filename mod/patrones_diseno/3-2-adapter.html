3-2: Adapter

Què és el Patró Adapter?

El patró Adapter és un patró estructural que permet que classes amb interfícies incompatibles treballin juntes. Aquest patró actua com un pont entre dues interfícies incompatibles, convertint la interfície d'una classe en una altra que el client espera.

Quan utilitzar el Patró Adapter?

  • Quan necessites utilitzar una classe existent però la seva interfície no coincideix amb la que necessites.
  • Quan vols crear una classe reutilitzable que col·labori amb classes no relacionades o amb interfícies incompatibles.
  • Quan vols utilitzar diverses subclasses existents, però és impracticable adaptar la seva interfície mitjançant herència.

Tipus d'Adapter

Hi ha dos tipus principals d'Adapter:

  1. Object Adapter: Utilitza la composició per adaptar una interfície a una altra.
  2. Class Adapter: Utilitza l'herència per adaptar una interfície a una altra.

Implementació del Patró Adapter

Exemples de Codi

Object Adapter
// Interfície objectiu que el client espera
interface Target {
    void request();
}

// Classe existent amb una interfície incompatible
class Adaptee {
    public void specificRequest() {
        System.out.println("Request específica");
    }
}

// Adapter que fa servir la composició
class Adapter implements Target {
    private Adaptee adaptee;

    public Adapter(Adaptee adaptee) {
        this.adaptee = adaptee;
    }

    @Override
    public void request() {
        adaptee.specificRequest();
    }
}

// Client
public class Client {
    public static void main(String[] args) {
        Adaptee adaptee = new Adaptee();
        Target adapter = new Adapter(adaptee);
        adapter.request(); // Sortida: Request específica
    }
}
Class Adapter
// Interfície objectiu que el client espera
interface Target {
    void request();
}

// Classe existent amb una interfície incompatible
class Adaptee {
    public void specificRequest() {
        System.out.println("Request específica");
    }
}

// Adapter que fa servir l'herència
class Adapter extends Adaptee implements Target {
    @Override
    public void request() {
        specificRequest();
    }
}

// Client
public class Client {
    public static void main(String[] args) {
        Target adapter = new Adapter();
        adapter.request(); // Sortida: Request específica
    }
}

Avantatges i Desavantatges

Avantatges

  • Reutilització de codi existent: Permet utilitzar classes existents sense modificar-les.
  • Flexibilitat: Facilita la integració de classes amb interfícies incompatibles.
  • Simplicitat: Simplifica el codi del client, ja que aquest només interactua amb la interfície objectiu.

Desavantatges

  • Complexitat: Pot afegir complexitat al sistema, especialment si es fa servir en excés.
  • Rendiment: Pot introduir una lleugera penalització de rendiment degut a la capa addicional d'abstracció.

Resum

El patró Adapter és una solució elegant per fer que classes amb interfícies incompatibles treballin juntes. Es pot implementar mitjançant composició (Object Adapter) o herència (Class Adapter). Aquest patró és útil quan es vol reutilitzar codi existent sense modificar-lo, però cal tenir en compte que pot afegir complexitat al sistema.
