Composite

Introducció al Patró Composite

El patró Composite és un patró estructural que permet tractar objectes individuals i composicions d'objectes de manera uniforme. Aquest patró és especialment útil quan es vol representar jerarquies d'objectes en forma d'arbre.

Objectius del Patró Composite

  • Uniformitat: Tractar objectes simples i compostos de la mateixa manera.
  • Flexibilitat: Facilitar l'addició de nous tipus de components.
  • Simplicitat: Reduir la complexitat del codi client que utilitza la jerarquia d'objectes.

Components del Patró Composite

  1. Component: Defineix la interfície per als objectes de la composició.
  2. Leaf: Representa els objectes individuals de la composició.
  3. Composite: Representa els objectes compostos que poden tenir altres components com a fills.

Diagrama UML del Patró Composite

Component
  + operation()
  + add(Component)
  + remove(Component)
  + getChild(int)

Leaf
  + operation()

Composite
  + operation()
  + add(Component)
  + remove(Component)
  + getChild(int)

Implementació del Patró Composite

A continuació es mostra una implementació del patró Composite en Java:

// Component
interface Graphic {
    void draw();
}

// Leaf
class Circle implements Graphic {
    @Override
    public void draw() {
        System.out.println("Drawing Circle");
    }
}

// Leaf
class Square implements Graphic {
    @Override
    public void draw() {
        System.out.println("Drawing Square");
    }
}

// Composite
class CompositeGraphic implements Graphic {
    private List<Graphic> childGraphics = new ArrayList<>();

    @Override
    public void draw() {
        for (Graphic graphic : childGraphics) {
            graphic.draw();
        }
    }

    public void add(Graphic graphic) {
        childGraphics.add(graphic);
    }

    public void remove(Graphic graphic) {
        childGraphics.remove(graphic);
    }

    public Graphic getChild(int index) {
        return childGraphics.get(index);
    }
}

// Client
public class CompositePatternDemo {
    public static void main(String[] args) {
        CompositeGraphic composite = new CompositeGraphic();
        composite.add(new Circle());
        composite.add(new Square());

        CompositeGraphic anotherComposite = new CompositeGraphic();
        anotherComposite.add(new Circle());
        anotherComposite.add(composite);

        anotherComposite.draw();
    }
}

Explicació del Codi

  1. Component: La interfície Graphic defineix l'operació draw().
  2. Leaf: Les classes Circle i Square implementen la interfície Graphic i defineixen el seu propi comportament per a draw().
  3. Composite: La classe CompositeGraphic implementa la interfície Graphic i manté una llista de components fills. Implementa les operacions add(), remove() i getChild(), a més de la seva pròpia implementació de draw(), que crida draw() en tots els seus fills.
  4. Client: La classe CompositePatternDemo crea una jerarquia d'objectes Graphic i crida draw() en el composite més gran, que a la seva vegada crida draw() en tots els seus components fills.

Avantatges i Desavantatges del Patró Composite

Avantatges:

  • Simplicitat: Facilita el tractament uniforme d'objectes individuals i compostos.
  • Escalabilitat: Permet afegir nous tipus de components fàcilment.
  • Flexibilitat: Facilita la manipulació de jerarquies complexes d'objectes.

Desavantatges:

  • Complexitat: Pot afegir complexitat al sistema si no es necessita una jerarquia d'objectes.
  • Rendiment: Pot afectar el rendiment si la jerarquia és molt profunda o si es fan moltes operacions de navegació.

Conclusió

El patró Composite és una solució elegant per tractar jerarquies d'objectes de manera uniforme. Permet als desenvolupadors crear estructures d'objectes complexes i manipular-les de manera senzilla i coherent. Tot i que pot afegir certa complexitat al sistema, els seus avantatges en termes de flexibilitat i escalabilitat sovint superen els desavantatges.
