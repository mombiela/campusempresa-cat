Mòdul 4: Patrons de Comportament

State

El patró State és un patró de disseny de comportament que permet a un objecte canviar el seu comportament quan el seu estat intern canvia. Aquest patró és especialment útil quan un objecte ha de canviar el seu comportament en funció del seu estat, i aquest canvi no ha de ser gestionat per múltiples condicions if-else o switch.

Conceptes Clau

  • Context: L'objecte que conté l'estat actual i delega el comportament a l'estat.
  • State: Una interfície que defineix el comportament associat a un estat particular del Context.
  • Concrete States: Implementacions concretes de la interfície State que defineixen el comportament específic per a cada estat.

Diagrama UML

+----------------+       +------------------+
|    Context     |       |      State       |
|----------------|       |------------------|
| - state: State |<>---->| + handle(): void |
| + request()    |       +------------------+
+----------------+               /_\
       |                          |
       |                          |
       |          +---------------------------+
       |          |                           |
+----------------+  +----------------+  +----------------+
| ConcreteStateA |  | ConcreteStateB |  | ConcreteStateC |
|----------------|  |----------------|  |----------------|
| + handle(): void|  | + handle(): void|  | + handle(): void|
+----------------+  +----------------+  +----------------+

Implementació en Java

A continuació es mostra un exemple de com implementar el patró State en Java.

// Interfície State
interface State {
    void handle(Context context);
}

// Implementació concreta de State
class ConcreteStateA implements State {
    @Override
    public void handle(Context context) {
        System.out.println("Handling request in State A");
        context.setState(new ConcreteStateB());
    }
}

class ConcreteStateB implements State {
    @Override
    public void handle(Context context) {
        System.out.println("Handling request in State B");
        context.setState(new ConcreteStateC());
    }
}

class ConcreteStateC implements State {
    @Override
    public void handle(Context context) {
        System.out.println("Handling request in State C");
        context.setState(new ConcreteStateA());
    }
}

// Context que conté l'estat actual
class Context {
    private State state;

    public Context(State state) {
        this.state = state;
    }

    public void setState(State state) {
        this.state = state;
    }

    public void request() {
        state.handle(this);
    }
}

// Exemple d'ús
public class StatePatternDemo {
    public static void main(String[] args) {
        Context context = new Context(new ConcreteStateA());
        
        context.request(); // Handling request in State A
        context.request(); // Handling request in State B
        context.request(); // Handling request in State C
        context.request(); // Handling request in State A
    }
}

Explicació del Codi

  1. Interfície State: Defineix el mètode handle que serà implementat per cada estat concret.
  2. Concrete States: ConcreteStateA, ConcreteStateB, i ConcreteStateC implementen el mètode handle i defineixen el comportament específic per a cada estat. Cada estat també decideix quin serà el següent estat.
  3. Context: Manté una referència a l'objecte State actual i delega les peticions a aquest objecte. També permet canviar l'estat actual.
  4. StatePatternDemo: Demostra com el Context canvia el seu comportament a mesura que canvia el seu estat.

Avantatges

  • Simplificació del codi: Elimina la necessitat de múltiples condicions if-else o switch.
  • Facilitat de manteniment: Cada estat es gestiona en una classe separada, facilitant la modificació i l'extensió del comportament.
  • Flexibilitat: Permet canviar el comportament d'un objecte en temps d'execució.

Desavantatges

  • Complexitat: Pot augmentar la complexitat del codi si hi ha molts estats.
  • Sobrecàrrega: Pot introduir una sobrecàrrega addicional en termes de nombre de classes.

Resum

El patró State és una solució elegant per gestionar el comportament d'un objecte basat en el seu estat intern. En lloc d'utilitzar múltiples condicions if-else o switch, el patró State permet encapsular cada estat en una classe separada, facilitant la gestió i l'extensió del comportament. Tot i que pot augmentar la complexitat del codi, els avantatges en termes de mantenibilitat i flexibilitat sovint superen els desavantatges.
