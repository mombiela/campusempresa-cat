4-10: Strategy

Què és el Patró Strategy?

El patró Strategy és un patró de disseny de comportament que permet definir una família d'algoritmes, encapsular-los i fer-los intercanviables. Aquest patró permet que l'algoritme variï independentment dels clients que l'utilitzen.

Objectius del Patró Strategy

  • Encapsulació d'algoritmes: Permet encapsular diferents algoritmes dins de classes separades.
  • Intercanviabilitat: Facilita la substitució d'un algoritme per un altre sense canviar el codi del client.
  • Flexibilitat: Permet afegir nous algoritmes fàcilment sense modificar el codi existent.

Components del Patró Strategy

  1. Context: Manté una referència a un objecte Strategy.
  2. Strategy: Interfície comuna per a tots els algoritmes.
  3. ConcreteStrategy: Implementacions concretes de l'interfície Strategy.

Exemple de Codi

Suposem que tenim una aplicació que calcula el preu de descompte de diferents tipus de productes. Podem utilitzar el patró Strategy per encapsular els diferents algoritmes de descompte.

Interfície Strategy

public interface DiscountStrategy {
    double applyDiscount(double price);
}

Implementacions Concretes de Strategy

public class NoDiscountStrategy implements DiscountStrategy {
    @Override
    public double applyDiscount(double price) {
        return price;
    }
}

public class SeasonalDiscountStrategy implements DiscountStrategy {
    @Override
    public double applyDiscount(double price) {
        return price * 0.9; // 10% de descompte
    }
}

public class ClearanceDiscountStrategy implements DiscountStrategy {
    @Override
    public double applyDiscount(double price) {
        return price * 0.5; // 50% de descompte
    }
}

Classe Context

public class Product {
    private String name;
    private double price;
    private DiscountStrategy discountStrategy;

    public Product(String name, double price) {
        this.name = name;
        this.price = price;
    }

    public void setDiscountStrategy(DiscountStrategy discountStrategy) {
        this.discountStrategy = discountStrategy;
    }

    public double getPrice() {
        return discountStrategy.applyDiscount(price);
    }

    @Override
    public String toString() {
        return "Product{" +
                "name='" + name + '\'' +
                ", price=" + price +
                '}';
    }
}

Utilització del Patró Strategy

public class Main {
    public static void main(String[] args) {
        Product product = new Product("Laptop", 1000.0);

        // Sense descompte
        product.setDiscountStrategy(new NoDiscountStrategy());
        System.out.println("Preu sense descompte: " + product.getPrice());

        // Descompte de temporada
        product.setDiscountStrategy(new SeasonalDiscountStrategy());
        System.out.println("Preu amb descompte de temporada: " + product.getPrice());

        // Descompte de liquidació
        product.setDiscountStrategy(new ClearanceDiscountStrategy());
        System.out.println("Preu amb descompte de liquidació: " + product.getPrice());
    }
}

Avantatges del Patró Strategy

  • Facilita l'extensibilitat: Afegir nous algoritmes és senzill i no requereix canvis en el codi existent.
  • Redueix la complexitat: Encapsula els algoritmes en classes separades, fent el codi més net i mantenible.
  • Permet la reutilització: Els algoritmes encapsulats poden ser reutilitzats en diferents contextos.

Desavantatges del Patró Strategy

  • Augmenta el nombre de classes: Cada algoritme requereix una nova classe, el que pot augmentar la complexitat del projecte.
  • Pot ser menys eficient: Si els algoritmes són molt senzills, la sobrecàrrega de crear classes addicionals pot ser innecessària.

Conclusió

El patró Strategy és una eina poderosa per gestionar la variabilitat dels algoritmes en el codi. Permet encapsular diferents comportaments i intercanviar-los fàcilment, millorant la flexibilitat i mantenibilitat del sistema. Tot i que pot augmentar el nombre de classes, els avantatges en termes de neteja i extensibilitat del codi solen superar aquest desavantatge.
