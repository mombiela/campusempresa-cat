3-6: Facade

Què és el Patró Facade?

El patró Facade és un patró estructural que proporciona una interfície simplificada a un conjunt complex de classes, llibreries o frameworks. Aquest patró és útil per reduir la complexitat i facilitar l'ús d'un sistema.

Objectius del Patró Facade

  • Simplificar la interfície d'un subsistema complex.
  • Reduir la dependència entre el client i les classes internes del subsistema.
  • Facilitar l'ús i la comprensió del sistema per part dels usuaris.

Estructura del Patró Facade

El patró Facade es compon de les següents parts:

  1. Facade: Proporciona una interfície simplificada per a les funcionalitats del subsistema.
  2. Subsistema: Conjunt de classes complexes que realitzen les operacions reals.

Diagrama UML

+-----------------+        +-----------------+
|     Client      |        |     Facade      |
+-----------------+        +-----------------+
        |                          |
        |                          |
        v                          v
+-----------------+        +-----------------+
|  Subsystem1     |        |  Subsystem2     |
+-----------------+        +-----------------+

Exemple de Codi

Suposem que tenim un sistema de subscripció a notícies amb diverses classes per gestionar diferents tipus de notícies. Implementarem un Facade per simplificar la interfície d'aquest sistema.

Classes del Subsistema

class SportsNews:
    def get_latest_news(self):
        return "Latest sports news"

class WeatherNews:
    def get_latest_news(self):
        return "Latest weather news"

class PoliticsNews:
    def get_latest_news(self):
        return "Latest politics news"

Classe Facade

class NewsFacade:
    def __init__(self):
        self.sports_news = SportsNews()
        self.weather_news = WeatherNews()
        self.politics_news = PoliticsNews()

    def get_all_news(self):
        return {
            "sports": self.sports_news.get_latest_news(),
            "weather": self.weather_news.get_latest_news(),
            "politics": self.politics_news.get_latest_news()
        }

Ús del Facade

if __name__ == "__main__":
    news_facade = NewsFacade()
    all_news = news_facade.get_all_news()
    for category, news in all_news.items():
        print(f"{category.capitalize()} News: {news}")

Avantatges del Patró Facade

  • Simplificació: Proporciona una interfície simplificada per a un subsistema complex.
  • Desacoblament: Redueix la dependència entre el client i les classes internes del subsistema.
  • Facilitat d'ús: Facilita l'ús i la comprensió del sistema per part dels usuaris.

Desavantatges del Patró Facade

  • Sobrecàrrega: Pot afegir una capa addicional de complexitat si no es dissenya correctament.
  • Limitació: Pot limitar l'accés a funcionalitats avançades del subsistema.

Conclusió

El patró Facade és una eina poderosa per simplificar la interfície d'un subsistema complex, reduint la dependència entre el client i les classes internes del subsistema. Tot i que pot afegir una capa addicional de complexitat, els avantatges en termes de facilitat d'ús i mantenibilitat sovint superen els desavantatges. Utilitzar el patró Facade pot fer que els sistemes siguin més accessibles i comprensibles per als desenvolupadors i usuaris finals.
