Mòdul 4: Patrons de Comportament

Iterator

Què és el Patró Iterator?

El patró Iterator és un patró de disseny de comportament que permet recórrer elements d'una col·lecció sense exposar la seva representació subjacent (llista, pila, arbre, etc.). Aquest patró proporciona una manera d'accedir als elements d'un objecte agregat seqüencialment sense exposar la seva estructura interna.

Objectius del Patró Iterator

  • Separació de Responsabilitats: Separa la lògica de navegació de la col·lecció de la lògica de la col·lecció en si.
  • Uniformitat: Proporciona una interfície uniforme per recórrer diferents tipus de col·leccions.
  • Flexibilitat: Permet canviar la col·lecció subjacent sense canviar el codi que la recorre.

Components del Patró Iterator

  1. Iterator: Defineix una interfície per accedir i recórrer els elements.
  2. ConcreteIterator: Implementa la interfície Iterator i manté la posició actual en la recórrer.
  3. Aggregate: Defineix una interfície per crear un Iterator.
  4. ConcreteAggregate: Implementa la interfície Aggregate i retorna una instància de ConcreteIterator.

Exemple de Codi

A continuació es mostra un exemple de com implementar el patró Iterator en Python:

from collections.abc import Iterable, Iterator

class ConcreteIterator(Iterator):
    def __init__(self, collection):
        self._collection = collection
        self._index = 0

    def __next__(self):
        try:
            value = self._collection[self._index]
            self._index += 1
            return value
        except IndexError:
            raise StopIteration

class ConcreteAggregate(Iterable):
    def __init__(self):
        self._items = []

    def __iter__(self):
        return ConcreteIterator(self._items)

    def add_item(self, item):
        self._items.append(item)

# Ús del patró Iterator
collection = ConcreteAggregate()
collection.add_item(1)
collection.add_item(2)
collection.add_item(3)

for item in collection:
    print(item)

Explicació del Codi

  1. ConcreteIterator: Implementa la interfície Iterator i defineix el mètode __next__ per recórrer els elements de la col·lecció.
  2. ConcreteAggregate: Implementa la interfície Iterable i defineix el mètode __iter__ per retornar una instància de ConcreteIterator.
  3. Ús del patró Iterator: Es crea una instància de ConcreteAggregate, s'afegeixen elements i es recorre la col·lecció utilitzant un bucle for.

Avantatges del Patró Iterator

  • Simplicitat: Simplifica el codi client al proporcionar una manera uniforme de recórrer col·leccions.
  • Flexibilitat: Permet canviar la col·lecció subjacent sense afectar el codi que la recorre.
  • Encapsulació: No exposa la representació interna de la col·lecció.

Desavantatges del Patró Iterator

  • Sobrecàrrega: Pot introduir una sobrecàrrega addicional en termes de memòria i rendiment.
  • Complexitat: Pot augmentar la complexitat del codi en col·leccions molt grans o complexes.

Conclusió

El patró Iterator és una eina poderosa per recórrer col·leccions de manera uniforme i flexible. Encara que pot introduir una certa sobrecàrrega, els seus avantatges en termes de simplicitat i encapsulació sovint superen els inconvenients. És especialment útil en aplicacions on es treballa amb diferents tipus de col·leccions i es necessita una manera consistent d'accedir als seus elements.
