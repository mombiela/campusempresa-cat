5-5 Exercicis Pràctics

En aquest tema, es presentaran una sèrie d'exercicis pràctics per ajudar-te a aplicar els patrons de disseny que has après al llarg del curs. Aquests exercicis estan dissenyats per reforçar els teus coneixements i proporcionar-te experiència pràctica en la implementació de patrons de disseny en situacions reals.

Exercici 1: Implementació del Patró Singleton

Descripció: Implementa el patró Singleton per assegurar-te que només hi hagi una instància d'una classe en tota l'aplicació.

Requisits:

  1. Crea una classe Logger que només permeti una única instància.
  2. La classe ha de tenir un mètode log(message: String) que imprimeixi el missatge amb un timestamp.

Exemple de codi:

public class Logger {
    private static Logger instance;
    
    private Logger() {
        // Constructor privat per evitar la instanciació externa
    }
    
    public static Logger getInstance() {
        if (instance == null) {
            instance = new Logger();
        }
        return instance;
    }
    
    public void log(String message) {
        System.out.println("[" + System.currentTimeMillis() + "] " + message);
    }
}

// Ús del Singleton
public class Main {
    public static void main(String[] args) {
        Logger logger = Logger.getInstance();
        logger.log("Aquest és un missatge de log.");
    }
}

Exercici 2: Implementació del Patró Factory Method

Descripció: Implementa el patró Factory Method per crear objectes sense especificar la classe exacta de l'objecte que serà creat.

Requisits:

  1. Crea una interfície Transport amb un mètode deliver().
  2. Implementa dues classes concretes Truck i Ship que implementin la interfície Transport.
  3. Crea una classe TransportFactory amb un mètode createTransport(String type) que retorni una instància de Truck o Ship segons el tipus especificat.

Exemple de codi:

interface Transport {
    void deliver();
}

class Truck implements Transport {
    public void deliver() {
        System.out.println("Delivery by truck.");
    }
}

class Ship implements Transport {
    public void deliver() {
        System.out.println("Delivery by ship.");
    }
}

class TransportFactory {
    public Transport createTransport(String type) {
        if (type.equalsIgnoreCase("truck")) {
            return new Truck();
        } else if (type.equalsIgnoreCase("ship")) {
            return new Ship();
        }
        throw new IllegalArgumentException("Unknown transport type.");
    }
}

// Ús del Factory Method
public class Main {
    public static void main(String[] args) {
        TransportFactory factory = new TransportFactory();
        Transport transport = factory.createTransport("truck");
        transport.deliver();
    }
}

Exercici 3: Implementació del Patró Observer

Descripció: Implementa el patró Observer per permetre que un objecte notifiqui a altres objectes quan el seu estat canviï.

Requisits:

  1. Crea una interfície Observer amb un mètode update().
  2. Crea una classe Subject amb mètodes per afegir, eliminar i notificar observadors.
  3. Implementa una classe ConcreteObserver que implementi la interfície Observer.

Exemple de codi:

import java.util.ArrayList;
import java.util.List;

interface Observer {
    void update(String message);
}

class Subject {
    private List<Observer> observers = new ArrayList<>();
    
    public void addObserver(Observer observer) {
        observers.add(observer);
    }
    
    public void removeObserver(Observer observer) {
        observers.remove(observer);
    }
    
    public void notifyObservers(String message) {
        for (Observer observer : observers) {
            observer.update(message);
        }
    }
}

class ConcreteObserver implements Observer {
    private String name;
    
    public ConcreteObserver(String name) {
        this.name = name;
    }
    
    public void update(String message) {
        System.out.println(name + " received message: " + message);
    }
}

// Ús del Patró Observer
public class Main {
    public static void main(String[] args) {
        Subject subject = new Subject();
        
        Observer observer1 = new ConcreteObserver("Observer 1");
        Observer observer2 = new ConcreteObserver("Observer 2");
        
        subject.addObserver(observer1);
        subject.addObserver(observer2);
        
        subject.notifyObservers("Hello Observers!");
    }
}

Exercici 4: Implementació del Patró Decorator

Descripció: Implementa el patró Decorator per afegir funcionalitats addicionals a un objecte de manera dinàmica.

Requisits:

  1. Crea una interfície Component amb un mètode operation().
  2. Implementa una classe ConcreteComponent que implementi la interfície Component.
  3. Crea una classe abstracta Decorator que implementi la interfície Component i tingui una referència a un objecte Component.
  4. Implementa dues classes concretes ConcreteDecoratorA i ConcreteDecoratorB que heretin de Decorator i afegeixin funcionalitats addicionals.

Exemple de codi:

interface Component {
    void operation();
}

class ConcreteComponent implements Component {
    public void operation() {
        System.out.println("ConcreteComponent operation.");
    }
}

abstract class Decorator implements Component {
    protected Component component;
    
    public Decorator(Component component) {
        this.component = component;
    }
    
    public void operation() {
        component.operation();
    }
}

class ConcreteDecoratorA extends Decorator {
    public ConcreteDecoratorA(Component component) {
        super(component);
    }
    
    public void operation() {
        super.operation();
        System.out.println("ConcreteDecoratorA additional operation.");
    }
}

class ConcreteDecoratorB extends Decorator {
    public ConcreteDecoratorB(Component component) {
        super(component);
    }
    
    public void operation() {
        super.operation();
        System.out.println("ConcreteDecoratorB additional operation.");
    }
}

// Ús del Patró Decorator
public class Main {
    public static void main(String[] args) {
        Component component = new ConcreteComponent();
        Component decoratorA = new ConcreteDecoratorA(component);
        Component decoratorB = new ConcreteDecoratorB(decoratorA);
        
        decoratorB.operation();
    }
}

Conclusió

Aquests exercicis pràctics t'ajudaran a consolidar els teus coneixements sobre els patrons de disseny i a veure com es poden aplicar en situacions reals. Practicar la implementació d'aquests patrons et permetrà reconèixer millor quan i com utilitzar-los en els teus projectes de programari. Recorda que la clau per dominar els patrons de disseny és la pràctica constant i l'aplicació en projectes reals.
