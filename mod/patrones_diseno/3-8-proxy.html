3-8: Proxy

Què és el Patró Proxy?

El patró Proxy és un patró estructural que proporciona un substitut o representant per a un altre objecte per controlar l'accés a aquest. Aquest patró és útil quan necessitem controlar l'accés a un objecte, afegir funcionalitats addicionals abans o després de l'accés a l'objecte, o quan l'objecte original és costós de crear o consumir recursos.

Tipus de Proxies

  1. Proxy remot: Controla l'accés a un objecte que es troba en una ubicació remota.
  2. Proxy virtual: Retarda la creació i inicialització d'un objecte costós fins que sigui necessari.
  3. Proxy de protecció: Controla l'accés a un objecte basant-se en permisos.
  4. Proxy de memòria cau: Proporciona un emmagatzematge temporal dels resultats d'operacions costoses per millorar el rendiment.

Estructura del Patró Proxy

El patró Proxy es compon de tres elements principals:

  1. Subject: Defineix la interfície comuna per a RealSubject i Proxy.
  2. RealSubject: L'objecte real que el Proxy representa.
  3. Proxy: Manté una referència a l'objecte RealSubject i controla l'accés a aquest.

Diagrama UML

+-----------+       +-----------+
|   Client  |------>|   Proxy   |
+-----------+       +-----------+
                        |
                        v
                  +-----------+
                  | RealSubject|
                  +-----------+

Exemple de Codi

A continuació, es mostra un exemple de codi en Python que implementa el patró Proxy.

from abc import ABC, abstractmethod

# Subject
class Subject(ABC):
    @abstractmethod
    def request(self):
        pass

# RealSubject
class RealSubject(Subject):
    def request(self):
        print("RealSubject: Handling request.")

# Proxy
class Proxy(Subject):
    def __init__(self, real_subject: RealSubject):
        self._real_subject = real_subject

    def request(self):
        if self.check_access():
            self._real_subject.request()
            self.log_access()

    def check_access(self) -> bool:
        print("Proxy: Checking access prior to firing a real request.")
        return True

    def log_access(self):
        print("Proxy: Logging the time of request.")

# Client code
def client_code(subject: Subject):
    subject.request()

if __name__ == "__main__":
    print("Client: Executing the client code with a real subject:")
    real_subject = RealSubject()
    client_code(real_subject)

    print("\nClient: Executing the same client code with a proxy:")
    proxy = Proxy(real_subject)
    client_code(proxy)

Explicació del Codi

  1. Subject: Defineix una interfície abstracta request que serà implementada per RealSubject i Proxy.
  2. RealSubject: Implementa la interfície request i conté la lògica principal.
  3. Proxy: Implementa la interfície request, controla l'accés a RealSubject i afegeix funcionalitats addicionals com check_access i log_access.
  4. Client code: Utilitza tant RealSubject com Proxy de manera intercanviable gràcies a la interfície comuna Subject.

Avantatges i Desavantatges del Patró Proxy

Avantatges:

  • Controla l'accés a l'objecte real.
  • Pot afegir funcionalitats addicionals sense modificar l'objecte real.
  • Pot millorar el rendiment retardant la creació de l'objecte real fins que sigui necessari.

Desavantatges:

  • Pot afegir complexitat addicional al sistema.
  • Pot introduir latència si no es gestiona correctament.

Conclusió

El patró Proxy és una eina poderosa per controlar l'accés a objectes i afegir funcionalitats addicionals de manera transparent. És especialment útil en situacions on l'objecte real és costós de crear o quan es necessita un control estricte d'accés. Tot i això, és important utilitzar-lo amb cura per evitar afegir complexitat innecessària al sistema.
