2-6: Prototype

Què és el Patró Prototype?

El patró Prototype és un patró creacional que permet copiar objectes existents sense necessitat de dependre de les seves classes concretes. Aquest patró és útil quan la creació d'un objecte és costosa o complexa, i es vol evitar la creació repetida d'objectes similars.

Conceptes Clau

  • Clonació: Crear una còpia exacta d'un objecte existent.
  • Prototip: L'objecte que es clona.
  • Interfície Prototype: Defineix el mètode de clonació.

Avantatges del Patró Prototype

  1. Reducció de la Complexitat: Simplifica la creació d'objectes complexos.
  2. Millora del Rendiment: Evita la creació repetida d'objectes costosos.
  3. Flexibilitat: Permet la creació d'objectes en temps d'execució.

Desavantatges del Patró Prototype

  1. Còpia Profunda vs. Còpia Superficial: Pot ser complicat implementar una còpia profunda.
  2. Gestió de Recursos: Pot ser difícil gestionar recursos com connexions de xarxa o fitxers oberts.

Implementació del Patró Prototype

Interfície Prototype

public interface Prototype {
    Prototype clone();
}

Classe Concreta que Implementa Prototype

public class ConcretePrototype implements Prototype {
    private String field;

    public ConcretePrototype(String field) {
        this.field = field;
    }

    @Override
    public Prototype clone() {
        return new ConcretePrototype(this.field);
    }

    public String getField() {
        return field;
    }

    public void setField(String field) {
        this.field = field;
    }
}

Ús del Patró Prototype

public class PrototypeDemo {
    public static void main(String[] args) {
        ConcretePrototype prototype1 = new ConcretePrototype("Original");
        ConcretePrototype prototype2 = (ConcretePrototype) prototype1.clone();

        System.out.println("Prototype 1 Field: " + prototype1.getField());
        System.out.println("Prototype 2 Field: " + prototype2.getField());

        prototype2.setField("Cloned");
        System.out.println("Prototype 1 Field after change: " + prototype1.getField());
        System.out.println("Prototype 2 Field after change: " + prototype2.getField());
    }
}

Explicació del Codi

  1. Interfície Prototype: Defineix el mètode clone() que serà implementat per les classes concretes.
  2. Classe ConcretePrototype: Implementa la interfície Prototype i defineix el mètode clone() per crear una còpia de si mateixa.
  3. PrototypeDemo: Demostra com es pot utilitzar el patró Prototype per clonar objectes. Es crea un objecte ConcretePrototype i es clona utilitzant el mètode clone(). Després es modifica el camp del clon per mostrar que són objectes independents.

Còpia Profunda vs. Còpia Superficial

  • Còpia Superficial: Copia els valors dels camps de l'objecte original al clon. Si els camps són referències a altres objectes, només es copien les referències.
  • Còpia Profunda: Copia els valors dels camps i també crea còpies dels objectes referenciats.

Exemple de Còpia Profunda

public class DeepPrototype implements Prototype {
    private List<String> list;

    public DeepPrototype(List<String> list) {
        this.list = new ArrayList<>(list);
    }

    @Override
    public Prototype clone() {
        return new DeepPrototype(new ArrayList<>(this.list));
    }

    public List<String> getList() {
        return list;
    }

    public void setList(List<String> list) {
        this.list = list;
    }
}

Conclusió

El patró Prototype és una eina poderosa per crear còpies d'objectes de manera eficient i flexible. És especialment útil quan la creació d'objectes és costosa o complexa. Tot i això, cal tenir cura amb la implementació de la clonació, especialment quan es treballa amb còpies profundes i la gestió de recursos.

Aquest patró és una part essencial del conjunt de patrons creacionals i proporciona una manera elegant de gestionar la creació d'objectes en aplicacions de programari.
