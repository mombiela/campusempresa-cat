Introducció als Patrons de Comportament

Què són els Patrons de Comportament?

Els patrons de comportament se centren en la manera com els objectes interactuen i es comuniquen entre ells. Aquests patrons ajuden a definir com els objectes col·laboren per acomplir tasques complexes, distribuint les responsabilitats de manera eficient i flexible.

Objectius dels Patrons de Comportament

  1. Simplificar la Comunicació: Facilitar la interacció entre objectes.
  2. Reduir la Complexitat: Minimitzar la complexitat del codi mitjançant la distribució de responsabilitats.
  3. Flexibilitat i Reutilització: Permetre que els objectes siguin més flexibles i reutilitzables.

Classificació dels Patrons de Comportament

Els patrons de comportament es poden classificar en diverses categories segons la seva funció principal:

  1. Patrons de Responsabilitat: Distribueixen les responsabilitats entre diferents classes.

    • Chain of Responsibility
    • Command
    • Interpreter

  2. Patrons de Comunicació: Faciliten la comunicació entre objectes.

    • Mediator
    • Observer

  3. Patrons de Control de Flux: Gestionen el flux de control en una aplicació.

    • Iterator
    • State
    • Strategy
    • Template Method

  4. Patrons de Gestió d'Estat: Gestionen l'estat intern dels objectes.

    • Memento
    • Visitor

Exemples de Patrons de Comportament

Chain of Responsibility

Aquest patró permet que diversos objectes tinguin l'oportunitat de gestionar una petició, evitant l'acoblament entre l'emissor i el receptor de la petició.

public abstract class Handler {
    protected Handler successor;

    public void setSuccessor(Handler successor) {
        this.successor = successor;
    }

    public abstract void handleRequest(Request request);
}

public class ConcreteHandler1 extends Handler {
    public void handleRequest(Request request) {
        if (request.getType() == RequestType.TYPE1) {
            // Process the request
        } else if (successor != null) {
            successor.handleRequest(request);
        }
    }
}

Observer

Aquest patró defineix una dependència un-a-molts entre objectes, de manera que quan un objecte canvia el seu estat, tots els seus dependents són notificats i actualitzats automàticament.

public interface Observer {
    void update();
}

public class ConcreteObserver implements Observer {
    public void update() {
        // React to the update
    }
}

public class Subject {
    private List<Observer> observers = new ArrayList<>();

    public void addObserver(Observer observer) {
        observers.add(observer);
    }

    public void notifyObservers() {
        for (Observer observer : observers) {
            observer.update();
        }
    }
}

Avantatges dels Patrons de Comportament

  1. Desacoblament: Redueixen l'acoblament entre objectes, facilitant el manteniment i l'extensió del codi.
  2. Reutilització: Promouen la reutilització de codi mitjançant la definició de comportaments comuns.
  3. Flexibilitat: Permeten canviar el comportament del sistema de manera dinàmica.

Desavantatges dels Patrons de Comportament

  1. Complexitat: Poden augmentar la complexitat del sistema, especialment en casos on s'utilitzen molts patrons simultàniament.
  2. Sobrecàrrega: La implementació de certs patrons pot introduir una sobrecàrrega de rendiment.

Conclusió

Els patrons de comportament són essencials per a la creació de sistemes flexibles i mantenibles. Aquests patrons faciliten la comunicació i la col·laboració entre objectes, distribuint les responsabilitats de manera eficient. En els següents temes, explorarem cada patró de comportament en detall, proporcionant exemples pràctics i exercicis per ajudar-vos a comprendre i aplicar aquests patrons en els vostres projectes de programari.
