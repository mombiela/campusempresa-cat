2-5: Builder

Què és el Patró Builder?

El patró Builder és un patró de disseny creacional que permet la construcció d'objectes complexos pas a pas. A diferència d'altres patrons creacionals que es centren en la instanciació d'objectes, el Builder se centra en la construcció d'objectes complexos, separant la seva representació de la seva construcció.

Quan Utilitzar el Patró Builder?

El patró Builder és especialment útil en els següents casos:

  • Quan l'objecte que es vol crear és complex i requereix molts passos per ser construït.
  • Quan es vol evitar un constructor amb molts paràmetres.
  • Quan es vol construir diferents representacions d'un mateix objecte.

Components del Patró Builder

  1. Builder: Defineix una interfície abstracta per a la creació de les parts d'un objecte Producte.
  2. ConcreteBuilder: Implementa la interfície Builder i construeix les parts de l'objecte Producte.
  3. Product: Representa l'objecte complex que es vol construir.
  4. Director: Construeix un objecte utilitzant l'interfície Builder.

Exemple de Codi

A continuació, es mostra un exemple del patró Builder en Java:

// Producte
public class House {
    private String foundation;
    private String structure;
    private String roof;
    private String interior;

    // Getters i Setters
    public void setFoundation(String foundation) { this.foundation = foundation; }
    public void setStructure(String structure) { this.structure = structure; }
    public void setRoof(String roof) { this.roof = roof; }
    public void setInterior(String interior) { this.interior = interior; }

    @Override
    public String toString() {
        return "House [foundation=" + foundation + ", structure=" + structure + ", roof=" + roof + ", interior=" + interior + "]";
    }
}

// Builder
public interface HouseBuilder {
    void buildFoundation();
    void buildStructure();
    void buildRoof();
    void buildInterior();
    House getHouse();
}

// ConcreteBuilder
public class ConcreteHouseBuilder implements HouseBuilder {
    private House house;

    public ConcreteHouseBuilder() {
        this.house = new House();
    }

    @Override
    public void buildFoundation() {
        house.setFoundation("Concrete Foundation");
        System.out.println("ConcreteHouseBuilder: Foundation complete...");
    }

    @Override
    public void buildStructure() {
        house.setStructure("Concrete Structure");
        System.out.println("ConcreteHouseBuilder: Structure complete...");
    }

    @Override
    public void buildRoof() {
        house.setRoof("Concrete Roof");
        System.out.println("ConcreteHouseBuilder: Roof complete...");
    }

    @Override
    public void buildInterior() {
        house.setInterior("Concrete Interior");
        System.out.println("ConcreteHouseBuilder: Interior complete...");
    }

    @Override
    public House getHouse() {
        return this.house;
    }
}

// Director
public class ConstructionEngineer {
    private HouseBuilder houseBuilder;

    public ConstructionEngineer(HouseBuilder houseBuilder) {
        this.houseBuilder = houseBuilder;
    }

    public House constructHouse() {
        this.houseBuilder.buildFoundation();
        this.houseBuilder.buildStructure();
        this.houseBuilder.buildRoof();
        this.houseBuilder.buildInterior();
        return this.houseBuilder.getHouse();
    }
}

// Client
public class BuilderPatternTest {
    public static void main(String[] args) {
        HouseBuilder concreteHouseBuilder = new ConcreteHouseBuilder();
        ConstructionEngineer engineer = new ConstructionEngineer(concreteHouseBuilder);
        House house = engineer.constructHouse();
        System.out.println("House is: " + house);
    }
}

Explicació del Codi

  1. Producte (House): Aquesta classe representa l'objecte complex que es vol construir. Té diversos atributs com foundation, structure, roof i interior, amb els seus respectius getters i setters.

  2. Builder (HouseBuilder): Aquesta interfície defineix els mètodes necessaris per construir les parts de l'objecte House.

  3. ConcreteBuilder (ConcreteHouseBuilder): Aquesta classe implementa la interfície HouseBuilder i proporciona les implementacions concretes per construir les parts de l'objecte House.

  4. Director (ConstructionEngineer): Aquesta classe utilitza un objecte HouseBuilder per construir un objecte House pas a pas.

  5. Client (BuilderPatternTest): Aquesta classe crea un objecte ConcreteHouseBuilder i un objecte ConstructionEngineer, i utilitza aquest últim per construir una casa.

Avantatges del Patró Builder

  • Separació de la construcció i representació: Permet separar la lògica de construcció de l'objecte de la seva representació final.
  • Facilitat d'ús: Facilita la creació d'objectes complexos amb molts paràmetres.
  • Flexibilitat: Permet construir diferents representacions d'un mateix objecte.

Desavantatges del Patró Builder

  • Complexitat: Pot introduir complexitat addicional al codi, especialment si l'objecte a construir no és molt complex.
  • Més classes: Requereix la creació de múltiples classes addicionals, com ara el Builder i el Director.

Conclusió

El patró Builder és una eina poderosa per a la construcció d'objectes complexos de manera controlada i flexible. Tot i que pot introduir una certa complexitat addicional, els avantatges que ofereix en termes de separació de la construcció i representació, així com la facilitat d'ús, el fan una opció valuosa en molts escenaris de desenvolupament de programari.
