2-4: Abstract Factory

Què és el patró Abstract Factory?

El patró Abstract Factory és un patró de disseny creacional que proporciona una interfície per crear famílies d'objectes relacionats o dependents sense especificar les seves classes concretes. Aquest patró és útil quan el sistema ha de ser independent de com es creen, componen i representen els seus objectes.

Objectius del patró Abstract Factory

  • Independència de la implementació: Permet que el codi client sigui independent de les classes concretes que es creen.
  • Consistència: Assegura que els objectes creats siguin compatibles entre ells.
  • Extensibilitat: Facilita l'extensió del codi per afegir noves famílies de productes.

Components del patró Abstract Factory

  1. Abstract Factory: Declara una interfície per crear productes abstractes.
  2. Concrete Factory: Implementa la interfície de l'Abstract Factory per crear objectes concrets.
  3. Abstract Product: Declara una interfície per a un tipus d'objecte.
  4. Concrete Product: Implementa la interfície de l'Abstract Product.
  5. Client: Utilitza només les interfícies declarades per l'Abstract Factory i l'Abstract Product.

Exemple de codi

Suposem que estem desenvolupant una aplicació que ha de funcionar amb diferents tipus de botons i caixes de text per a diferents sistemes operatius (Windows i Mac).

Abstract Factory

public interface GUIFactory {
    Button createButton();
    TextBox createTextBox();
}

Concrete Factory

public class WindowsFactory implements GUIFactory {
    @Override
    public Button createButton() {
        return new WindowsButton();
    }

    @Override
    public TextBox createTextBox() {
        return new WindowsTextBox();
    }
}

public class MacFactory implements GUIFactory {
    @Override
    public Button createButton() {
        return new MacButton();
    }

    @Override
    public TextBox createTextBox() {
        return new MacTextBox();
    }
}

Abstract Product

public interface Button {
    void paint();
}

public interface TextBox {
    void render();
}

Concrete Product

public class WindowsButton implements Button {
    @Override
    public void paint() {
        System.out.println("Rendering a button in Windows style.");
    }
}

public class WindowsTextBox implements TextBox {
    @Override
    public void render() {
        System.out.println("Rendering a text box in Windows style.");
    }
}

public class MacButton implements Button {
    @Override
    public void paint() {
        System.out.println("Rendering a button in Mac style.");
    }
}

public class MacTextBox implements TextBox {
    @Override
    public void render() {
        System.out.println("Rendering a text box in Mac style.");
    }
}

Client

public class Application {
    private Button button;
    private TextBox textBox;

    public Application(GUIFactory factory) {
        button = factory.createButton();
        textBox = factory.createTextBox();
    }

    public void paint() {
        button.paint();
        textBox.render();
    }
}

public class ApplicationConfigurator {
    public static void main(String[] args) {
        GUIFactory factory;
        String osName = System.getProperty("os.name").toLowerCase();
        if (osName.contains("win")) {
            factory = new WindowsFactory();
        } else {
            factory = new MacFactory();
        }

        Application app = new Application(factory);
        app.paint();
    }
}

Avantatges del patró Abstract Factory

  • Encapsulació de la creació d'objectes: El client no necessita conèixer les classes concretes que es creen.
  • Facilitat de canvi: És fàcil canviar la família de productes utilitzada per l'aplicació.
  • Consistència: Assegura que els objectes creats per una fàbrica concreta siguin compatibles entre ells.

Desavantatges del patró Abstract Factory

  • Complexitat: Pot afegir complexitat al codi, especialment si només es necessita un nombre reduït de productes.
  • Dificultat d'extensió: Afegir nous productes pot requerir canvis a les interfícies de les fàbriques i els productes.

Resum

El patró Abstract Factory és una solució poderosa per gestionar la creació d'objectes en aplicacions que necessiten ser independents de les classes concretes que utilitzen. Encara que pot afegir complexitat, els avantatges en termes de consistència i extensibilitat sovint superen els desavantatges. Aquest patró és especialment útil en aplicacions que necessiten suportar múltiples famílies de productes o sistemes operatius.
