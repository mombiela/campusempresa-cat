3-5: Decorator

Què és el Patró Decorator?

El patró Decorator és un patró estructural que permet afegir funcionalitats addicionals a un objecte de manera dinàmica. Aquest patró és útil quan volem estendre les capacitats d'una classe sense modificar el seu codi original, seguint el principi de responsabilitat única i el principi de tancament obert.

Objectius del Patró Decorator

  • Flexibilitat: Permet afegir funcionalitats de manera dinàmica.
  • Reutilització: Facilita la reutilització de codi.
  • Simplicitat: Evita la creació de subclasses per cada combinació de funcionalitats.

Components del Patró Decorator

  1. Component: Interfície o classe abstracta que defineix les operacions que es poden realitzar.
  2. ConcreteComponent: Implementació concreta del component.
  3. Decorator: Classe abstracta que implementa la interfície del component i conté una referència a un objecte Component.
  4. ConcreteDecorator: Classes concretes que estenen la funcionalitat del component decorat.

Exemple de Codi

A continuació es mostra un exemple en Python per il·lustrar el patró Decorator:

from abc import ABC, abstractmethod

# Component
class Beverage(ABC):
    @abstractmethod
    def cost(self) -> float:
        pass

# ConcreteComponent
class Espresso(Beverage):
    def cost(self) -> float:
        return 1.99

# Decorator
class AddonDecorator(Beverage, ABC):
    def __init__(self, beverage: Beverage):
        self._beverage = beverage

# ConcreteDecorator
class Milk(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.50

class Sugar(AddonDecorator):
    def cost(self) -> float:
        return self._beverage.cost() + 0.20

# Ús del patró Decorator
if __name__ == "__main__":
    beverage = Espresso()
    print(f"Cost of Espresso: ${beverage.cost()}")

    beverage_with_milk = Milk(beverage)
    print(f"Cost of Espresso with Milk: ${beverage_with_milk.cost()}")

    beverage_with_milk_and_sugar = Sugar(beverage_with_milk)
    print(f"Cost of Espresso with Milk and Sugar: ${beverage_with_milk_and_sugar.cost()}")

Explicació del Codi

  1. Component: Beverage és una classe abstracta que defineix el mètode cost.
  2. ConcreteComponent: Espresso implementa el mètode cost retornant el preu de l'espresso.
  3. Decorator: AddonDecorator és una classe abstracta que conté una referència a un objecte Beverage.
  4. ConcreteDecorator: Milk i Sugar són decoradors concrets que afegeixen costos addicionals al preu de la beguda.

Avantatges del Patró Decorator

  • Flexibilitat: Permet afegir funcionalitats de manera dinàmica sense modificar el codi existent.
  • Composició: Facilita la combinació de múltiples decoradors per crear noves funcionalitats.
  • Simplicitat: Redueix la necessitat de crear subclasses per cada combinació de funcionalitats.

Desavantatges del Patró Decorator

  • Complexitat: Pot augmentar la complexitat del codi degut a la creació de moltes petites classes.
  • Depuració: Pot ser més difícil de depurar i rastrejar a causa de la composició dinàmica dels objectes.

Conclusió

El patró Decorator és una eina poderosa per afegir funcionalitats a objectes de manera dinàmica i flexible. És especialment útil quan es volen evitar subclasses redundants i es busca una solució que permeti la combinació de funcionalitats de manera modular. Tot i que pot augmentar la complexitat del codi, els avantatges en termes de flexibilitat i reutilització sovint superen els inconvenients.
