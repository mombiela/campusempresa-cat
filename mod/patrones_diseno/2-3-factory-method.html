Mòdul 2: Patrons Creacionals

Factory Method

Introducció al Patró Factory Method

El patró Factory Method és un patró de disseny creacional que proporciona una interfície per crear objectes en una superclasse, però permet que les subclasses alterin el tipus d'objectes que es crearan. Aquest patró és especialment útil quan la creació de l'objecte implica una lògica complexa o quan el tipus d'objecte a crear no es coneix fins al moment de l'execució.

Conceptes Clau

  • Superclasse i Subclasse: La superclasse defineix el mètode de fàbrica, mentre que les subclasses implementen aquest mètode per crear objectes específics.
  • Interfície de Creació: La interfície que defineix el mètode de fàbrica.
  • Producte: L'objecte que es crea mitjançant el mètode de fàbrica.

Estructura del Patró

  1. Producte: Defineix la interfície dels objectes que la fàbrica crea.
  2. Producte Concret: Implementa la interfície del Producte.
  3. Creador: Declara el mètode de fàbrica que retorna un objecte de tipus Producte.
  4. Creador Concret: Implementa el mètode de fàbrica per retornar una instància de Producte Concret.

Exemple de Codi

Suposem que estem desenvolupant una aplicació que necessita crear diferents tipus de documents (PDF, Word). Utilitzarem el patró Factory Method per gestionar la creació d'aquests documents.

from abc import ABC, abstractmethod

# Producte
class Document(ABC):
    @abstractmethod
    def render(self):
        pass

# Producte Concret
class PDFDocument(Document):
    def render(self):
        return "Rendering PDF Document"

class WordDocument(Document):
    def render(self):
        return "Rendering Word Document"

# Creador
class DocumentCreator(ABC):
    @abstractmethod
    def create_document(self) -> Document:
        pass

    def render_document(self):
        document = self.create_document()
        return document.render()

# Creador Concret
class PDFDocumentCreator(DocumentCreator):
    def create_document(self) -> Document:
        return PDFDocument()

class WordDocumentCreator(DocumentCreator):
    def create_document(self) -> Document:
        return WordDocument()

# Ús del patró
pdf_creator = PDFDocumentCreator()
print(pdf_creator.render_document())  # Output: Rendering PDF Document

word_creator = WordDocumentCreator()
print(word_creator.render_document())  # Output: Rendering Word Document

Avantatges del Patró Factory Method

  • Flexibilitat: Permet que les subclasses decideixin quin tipus d'objecte crear.
  • Desacoblament: Separa la creació d'objectes de la seva implementació.
  • Extensibilitat: Facilita l'addició de nous tipus de productes sense modificar el codi existent.

Desavantatges del Patró Factory Method

  • Complexitat: Pot augmentar la complexitat del codi, especialment si es tenen moltes subclasses.
  • Sobrecàrrega: Pot introduir una sobrecàrrega addicional si la lògica de creació és senzilla i no justifica l'ús del patró.

Conclusió

El patró Factory Method és una eina poderosa per gestionar la creació d'objectes en aplicacions complexes. Permet una major flexibilitat i extensibilitat, tot i que pot introduir una certa complexitat addicional. És important avaluar si els avantatges del patró justifiquen la seva implementació en el context específic del projecte.

Aquest patró és especialment útil en situacions on la creació d'objectes implica una lògica complexa o quan el tipus d'objecte a crear no es coneix fins al moment de l'execució.
