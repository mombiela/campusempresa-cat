Mòdul 5: Aplicació de Patrons de Disseny

Combinació de Patrons de Disseny

En aquest tema, explorarem com combinar diferents patrons de disseny per resoldre problemes complexos de desenvolupament de programari. La combinació de patrons pot proporcionar solucions més robustes i flexibles, aprofitant els punts forts de cada patró.

Conceptes Clau

  1. Sinergia de Patrons: Com els patrons poden treballar junts per millorar la solució.
  2. Compatibilitat: Quins patrons són més fàcils de combinar.
  3. Exemples Pràctics: Casos reals on s'han combinat patrons.
  4. Avantatges i Desavantatges: Beneficis i possibles complicacions de combinar patrons.

Sinergia de Patrons

Els patrons de disseny sovint es complementen entre si. Per exemple, un patró creacional pot ser utilitzat per crear objectes que després seran manipulats per un patró estructural o de comportament.

  • Singleton + Factory Method: El patró Singleton pot assegurar que només hi hagi una instància d'una fàbrica (Factory Method) que creï objectes.
  • Composite + Iterator: El patró Composite pot ser combinat amb l'Iterator per permetre la navegació a través d'una estructura d'arbres.

Compatibilitat

Alguns patrons són més fàcils de combinar que altres. Aquí hi ha una taula que mostra algunes combinacions comunes:

Patró 1 Patró 2 Compatibilitat Descripció
Singleton Factory Method Alta Singleton pot gestionar la instància única d'una fàbrica.
Composite Iterator Alta Iterator pot recórrer una estructura Composite.
Adapter Facade Mitjana Adapter pot ser utilitzat dins d'una Facade per simplificar interfícies.
Observer Mediator Alta Mediator pot gestionar les comunicacions entre observadors i subjectes.
Strategy Template Method Mitjana Strategy pot definir algorismes que Template Method pot utilitzar.

Exemples Pràctics

  1. Singleton + Factory Method
public class SingletonFactory {
    private static SingletonFactory instance;

    private SingletonFactory() {}

    public static SingletonFactory getInstance() {
        if (instance == null) {
            instance = new SingletonFactory();
        }
        return instance;
    }

    public Product createProduct(String type) {
        if (type.equals("A")) {
            return new ProductA();
        } else if (type.equals("B")) {
            return new ProductB();
        }
        return null;
    }
}
  1. Composite + Iterator
public interface Component {
    void operation();
}

public class Leaf implements Component {
    public void operation() {
        System.out.println("Leaf operation");
    }
}

public class Composite implements Component {
    private List<Component> children = new ArrayList<>();

    public void add(Component component) {
        children.add(component);
    }

    public void remove(Component component) {
        children.remove(component);
    }

    public void operation() {
        for (Component child : children) {
            child.operation();
        }
    }

    public Iterator<Component> createIterator() {
        return children.iterator();
    }
}

Avantatges i Desavantatges

Avantatges:

  • Flexibilitat: Les solucions es tornen més flexibles i adaptables.
  • Reutilització: Es poden reutilitzar solucions provades i testades.
  • Mantenibilitat: El codi és més fàcil de mantenir i entendre.

Desavantatges:

  • Complexitat: La combinació de patrons pot augmentar la complexitat del codi.
  • Sobrecàrrega: Pot haver-hi una sobrecàrrega en termes de rendiment i recursos.
  • Temps d'Aprenentatge: Requereix un temps addicional per aprendre i implementar correctament.

Conclusió

La combinació de patrons de disseny és una tècnica poderosa que pot millorar significativament la qualitat del programari. No obstant això, és important comprendre bé cada patró individualment abans d'intentar combinar-los. Amb una bona comprensió i pràctica, la combinació de patrons pot portar a solucions més eficients i mantenibles.
