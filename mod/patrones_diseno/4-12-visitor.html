4-12: Visitor

Què és el Patró Visitor?

El patró de disseny Visitor és un patró de comportament que permet afegir noves operacions a objectes d'una jerarquia de classes sense modificar les classes sobre les quals opera. Això es fa mitjançant la separació de l'algorisme de les estructures de dades sobre les quals opera.

Components del Patró Visitor

  1. Visitor: Declara una interfície per a una operació que visita elements de l'estructura d'objectes.
  2. ConcreteVisitor: Implementa les operacions definides a la interfície Visitor.
  3. Element: Declara una interfície per acceptar un visitant.
  4. ConcreteElement: Implementa l'operació accept() que pren un visitant com a argument.
  5. ObjectStructure: Pot enumerar els seus elements i proporcionar una interfície per permetre que un visitant visiti els seus elements.

Diagrama UML

+-------------------+       +-------------------+
|      Visitor      |       |   ConcreteVisitor |
+-------------------+       +-------------------+
| + visitElementA() |<------| + visitElementA() |
| + visitElementB() |       | + visitElementB() |
+-------------------+       +-------------------+
          ^                           ^
          |                           |
+-------------------+       +-------------------+
|      Element      |       |  ConcreteElement  |
+-------------------+       +-------------------+
| + accept(visitor) |------>| + accept(visitor) |
+-------------------+       | + operation()     |
                            +-------------------+

Exemple de Codi

Suposem que tenim una jerarquia de classes que representa diferents tipus de formes geomètriques, i volem afegir una operació per calcular l'àrea de cada forma sense modificar les classes de les formes.

from abc import ABC, abstractmethod

# Interfície Visitor
class Visitor(ABC):
    @abstractmethod
    def visit_circle(self, circle):
        pass

    @abstractmethod
    def visit_rectangle(self, rectangle):
        pass

# Interfície Element
class Shape(ABC):
    @abstractmethod
    def accept(self, visitor):
        pass

# ConcreteElement: Circle
class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def accept(self, visitor):
        visitor.visit_circle(self)

# ConcreteElement: Rectangle
class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def accept(self, visitor):
        visitor.visit_rectangle(self)

# ConcreteVisitor: AreaCalculator
class AreaCalculator(Visitor):
    def visit_circle(self, circle):
        area = 3.14 * circle.radius ** 2
        print(f"L'àrea del cercle és {area}")

    def visit_rectangle(self, rectangle):
        area = rectangle.width * rectangle.height
        print(f"L'àrea del rectangle és {area}")

# Ús del patró Visitor
shapes = [Circle(10), Rectangle(5, 3)]
area_calculator = AreaCalculator()

for shape in shapes:
    shape.accept(area_calculator)

Explicació del Codi

  1. Interfície Visitor: Defineix les operacions visit_circle i visit_rectangle que seran implementades per AreaCalculator.
  2. Interfície Element: Defineix el mètode accept que pren un visitant com a argument.
  3. ConcreteElement: Circle i Rectangle implementen el mètode accept per permetre que un visitant realitzi operacions sobre ells.
  4. ConcreteVisitor: AreaCalculator implementa les operacions visit_circle i visit_rectangle per calcular l'àrea de les formes.
  5. Ús del patró Visitor: Es crea una llista de formes i es passa un visitant AreaCalculator a cada forma per calcular la seva àrea.

Avantatges del Patró Visitor

  • Separació de Responsabilitats: Permet separar les operacions de les estructures de dades.
  • Facilitat d'Afegir Noves Operacions: És fàcil afegir noves operacions sense modificar les classes de les formes.

Desavantatges del Patró Visitor

  • Dificultat d'Afegir Noves Classes: Afegir noves classes a la jerarquia de formes pot requerir canvis en tots els visitants.
  • Complexitat: Pot augmentar la complexitat del codi, especialment si hi ha moltes classes i operacions.

Conclusió

El patró Visitor és una eina poderosa per afegir funcionalitats a una jerarquia de classes sense modificar les classes existents. És especialment útil quan es necessita aplicar diverses operacions a objectes d'una estructura complexa. Tot i això, cal tenir en compte els seus desavantatges, especialment pel que fa a la dificultat d'afegir noves classes a la jerarquia.
