5-4: Estudi de Casos

En aquest tema, explorarem com els patrons de disseny es poden aplicar en situacions reals a través d'estudis de casos. Aquests exemples pràctics ajudaran a entendre com els patrons poden resoldre problemes comuns en el desenvolupament de programari.

Estudi de Cas 1: Sistema de Gestió de Continguts (CMS)

Problema

Un sistema de gestió de continguts (CMS) necessita permetre als usuaris crear, editar i publicar articles. A més, el sistema ha de suportar diferents tipus de contingut com articles de text, imatges i vídeos.

Solució

Utilitzarem diversos patrons de disseny per resoldre aquest problema:

  1. Factory Method: Per crear diferents tipus de contingut (articles, imatges, vídeos).
  2. Composite: Per gestionar la jerarquia de continguts, permetent que els articles puguin contenir altres articles, imatges o vídeos.
  3. Observer: Per notificar els usuaris quan es publiquen nous continguts.

Implementació

Factory Method

// Producte
interface Content {
    void render();
}

// Producte Concret
class Article implements Content {
    public void render() {
        System.out.println("Rendering an article.");
    }
}

class Image implements Content {
    public void render() {
        System.out.println("Rendering an image.");
    }
}

class Video implements Content {
    public void render() {
        System.out.println("Rendering a video.");
    }
}

// Creator
abstract class ContentCreator {
    public abstract Content createContent();
}

// Creators Concrets
class ArticleCreator extends ContentCreator {
    public Content createContent() {
        return new Article();
    }
}

class ImageCreator extends ContentCreator {
    public Content createContent() {
        return new Image();
    }
}

class VideoCreator extends ContentCreator {
    public Content createContent() {
        return new Video();
    }
}

Composite

// Component
interface ContentComponent {
    void show();
}

// Fulla
class SimpleContent implements ContentComponent {
    private String name;

    public SimpleContent(String name) {
        this.name = name;
    }

    public void show() {
        System.out.println(name);
    }
}

// Composite
class CompositeContent implements ContentComponent {
    private List<ContentComponent> contents = new ArrayList<>();

    public void add(ContentComponent content) {
        contents.add(content);
    }

    public void remove(ContentComponent content) {
        contents.remove(content);
    }

    public void show() {
        for (ContentComponent content : contents) {
            content.show();
        }
    }
}

Observer

// Subjecte
class ContentPublisher {
    private List<Observer> observers = new ArrayList<>();

    public void addObserver(Observer observer) {
        observers.add(observer);
    }

    public void removeObserver(Observer observer) {
        observers.remove(observer);
    }

    public void notifyObservers() {
        for (Observer observer : observers) {
            observer.update();
        }
    }

    public void publishContent() {
        // Publicar contingut
        notifyObservers();
    }
}

// Observador
interface Observer {
    void update();
}

// Observador Concret
class User implements Observer {
    private String name;

    public User(String name) {
        this.name = name;
    }

    public void update() {
        System.out.println(name + " has been notified of new content.");
    }
}

Estudi de Cas 2: Sistema de Reserva d'Hotels

Problema

Un sistema de reserva d'hotels necessita gestionar les reserves, permetre la cerca d'hotels segons diferents criteris i aplicar descomptes en funció de diverses promocions.

Solució

Utilitzarem diversos patrons de disseny per resoldre aquest problema:

  1. Strategy: Per implementar diferents estratègies de cerca d'hotels.
  2. Decorator: Per aplicar descomptes i promocions a les reserves.
  3. Facade: Per proporcionar una interfície simplificada per als usuaris del sistema.

Implementació

Strategy

// Estratègia
interface SearchStrategy {
    List<Hotel> search(String location);
}

// Estratègies Concretes
class PriceSearchStrategy implements SearchStrategy {
    public List<Hotel> search(String location) {
        // Cerca hotels per preu
        return new ArrayList<>();
    }
}

class RatingSearchStrategy implements SearchStrategy {
    public List<Hotel> search(String location) {
        // Cerca hotels per valoració
        return new ArrayList<>();
    }
}

// Context
class HotelSearch {
    private SearchStrategy strategy;

    public void setStrategy(SearchStrategy strategy) {
        this.strategy = strategy;
    }

    public List<Hotel> search(String location) {
        return strategy.search(location);
    }
}

Decorator

// Component
interface Booking {
    double getCost();
    String getDescription();
}

// Component Concret
class BasicBooking implements Booking {
    public double getCost() {
        return 100.0;
    }

    public String getDescription() {
        return "Basic booking";
    }
}

// Decorador
abstract class BookingDecorator implements Booking {
    protected Booking decoratedBooking;

    public BookingDecorator(Booking booking) {
        this.decoratedBooking = booking;
    }

    public double getCost() {
        return decoratedBooking.getCost();
    }

    public String getDescription() {
        return decoratedBooking.getDescription();
    }
}

// Decoradors Concrets
class BreakfastDecorator extends BookingDecorator {
    public BreakfastDecorator(Booking booking) {
        super(booking);
    }

    public double getCost() {
        return super.getCost() + 20.0;
    }

    public String getDescription() {
        return super.getDescription() + ", with breakfast";
    }
}

class SpaDecorator extends BookingDecorator {
    public SpaDecorator(Booking booking) {
        super(booking);
    }

    public double getCost() {
        return super.getCost() + 50.0;
    }

    public String getDescription() {
        return super.getDescription() + ", with spa access";
    }
}

Facade

// Subsistemes
class HotelBookingSystem {
    public void bookHotel(String hotelName) {
        System.out.println("Booking hotel: " + hotelName);
    }
}

class PaymentSystem {
    public void processPayment(double amount) {
        System.out.println("Processing payment of: $" + amount);
    }
}

class NotificationSystem {
    public void sendConfirmation(String email) {
        System.out.println("Sending confirmation to: " + email);
    }
}

// Façana
class BookingFacade {
    private HotelBookingSystem hotelBookingSystem;
    private PaymentSystem paymentSystem;
    private NotificationSystem notificationSystem;

    public BookingFacade() {
        this.hotelBookingSystem = new HotelBookingSystem();
        this.paymentSystem = new PaymentSystem();
        this.notificationSystem = new NotificationSystem();
    }

    public void bookHotel(String hotelName, double amount, String email) {
        hotelBookingSystem.bookHotel(hotelName);
        paymentSystem.processPayment(amount);
        notificationSystem.sendConfirmation(email);
    }
}

Conclusió

Els estudis de casos presentats mostren com els patrons de disseny poden ser aplicats en situacions reals per resoldre problemes comuns en el desenvolupament de programari. Aquests exemples pràctics ajuden a entendre la importància i l'eficàcia dels patrons de disseny en la creació de solucions reutilitzables i mantenibles.
