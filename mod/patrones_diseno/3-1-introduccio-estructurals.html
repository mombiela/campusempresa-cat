Introducció als Patrons Estructurals

Els patrons estructurals són patrons de disseny que s'ocupen de la composició de classes i objectes per formar estructures més grans. Aquests patrons ajuden a garantir que les entitats del sistema estiguin ben organitzades i que les relacions entre elles siguin clares i eficients. Els patrons estructurals faciliten la definició de relacions entre objectes i classes, permetent la reutilització de codi i la millora de la mantenibilitat del sistema.

Objectius dels Patrons Estructurals

  • Simplificar la composició d'objectes i classes: Faciliten la creació de sistemes complexos a partir de components més senzills.
  • Millorar la reutilització del codi: Permeten reutilitzar components existents en nous contextos.
  • Facilitar la mantenibilitat: Ajuden a mantenir el codi net i organitzat, fent-lo més fàcil de mantenir i estendre.
  • Reduir la complexitat: Simplifiquen la interacció entre objectes i classes, reduint la complexitat del sistema.

Classificació dels Patrons Estructurals

Els patrons estructurals es poden classificar en dues categories principals:

  1. Patrons de Classe: Aquests patrons utilitzen l'herència per compondre interfícies o implementacions.
  2. Patrons d'Objecte: Aquests patrons utilitzen la composició d'objectes per definir noves funcionalitats.

Llista de Patrons Estructurals

En aquest mòdul, explorarem els següents patrons estructurals:

  1. Adapter: Converteix la interfície d'una classe en una altra interfície que els clients esperen.
  2. Bridge: Separa l'abstracció de la seva implementació perquè ambdues puguin variar independentment.
  3. Composite: Compon objectes en estructures d'arbre per representar jerarquies part-tot.
  4. Decorator: Afegeix funcionalitat addicional a un objecte de manera dinàmica.
  5. Facade: Proporciona una interfície simplificada a un conjunt de classes o a un subsistema complex.
  6. Flyweight: Utilitza el compartiment de dades per suportar grans quantitats d'objectes petits de manera eficient.
  7. Proxy: Proporciona un substitut o marcador de lloc per a un altre objecte per controlar l'accés a aquest.

Exemples de Patrons Estructurals

Exemple de Patron Adapter

El patró Adapter permet que classes amb interfícies incompatibles treballin juntes. A continuació es mostra un exemple en Python:

class Target:
    def request(self):
        return "Target: The default target's behavior."

class Adaptee:
    def specific_request(self):
        return ".eetpadA eht fo roivaheb laicepS"

class Adapter(Target, Adaptee):
    def request(self):
        return f"Adapter: (TRANSLATED) {self.specific_request()[::-1]}"

def client_code(target: Target):
    print(target.request())

if __name__ == "__main__":
    print("Client: I can work just fine with the Target objects:")
    target = Target()
    client_code(target)

    print("\nClient: The Adaptee class has a weird interface. See, I don't understand it:")
    adaptee = Adaptee()
    print(f"Adaptee: {adaptee.specific_request()}")

    print("\nClient: But I can work with it via the Adapter:")
    adapter = Adapter()
    client_code(adapter)

En aquest exemple:

  • Target: Defineix la interfície que el client espera.
  • Adaptee: Té una interfície incompatible amb el client.
  • Adapter: Converteix la interfície de l'Adaptee a una interfície compatible amb el Target.

Conclusió

Els patrons estructurals són essencials per a la construcció de sistemes de programari ben organitzats i mantenibles. Aquests patrons ajuden a definir relacions clares i eficients entre classes i objectes, facilitant la reutilització del codi i la reducció de la complexitat del sistema. En els següents temes, explorarem cada un dels patrons estructurals en detall, proporcionant exemples pràctics i exercicis per ajudar-vos a comprendre i aplicar aquests patrons en els vostres projectes de programari.
