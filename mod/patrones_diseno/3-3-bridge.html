3-3: Bridge

Què és el Patró Bridge?

El patró Bridge és un patró estructural que separa l'abstracció de la seva implementació perquè ambdues puguin variar independentment. Aquest patró és especialment útil quan es vol evitar una explosió de classes derivades en sistemes complexos.

Objectius del Patró Bridge

  • Separar l'abstracció de la implementació: Permet que l'abstracció i la implementació evolucionin independentment.
  • Reduir la complexitat: Evita la combinació de múltiples dimensions de variació en una sola jerarquia de classes.
  • Facilitar l'extensibilitat: Facilita l'extensió de les classes sense afectar les altres.

Estructura del Patró Bridge

El patró Bridge es compon de quatre components principals:

  1. Abstracció: Defineix la interfície abstracta i manté una referència a un objecte Implementador.
  2. Refinament de l'Abstracció: Extén la interfície de l'Abstracció.
  3. Implementador: Defineix la interfície per a les classes d'implementació.
  4. Concret Implementador: Proporciona la implementació concreta de la interfície Implementador.

Diagrama UML

Abstracció                    Implementador
+ operation()                 + operationImpl()
   |                               |
   v                               v
Refinament de l'Abstracció    Concret Implementador
+ operation()                 + operationImpl()

Exemple de Codi

Suposem que estem desenvolupant una aplicació que dibuixa formes (com cercles i quadrats) amb diferents colors. Utilitzarem el patró Bridge per separar la forma (abstracció) del color (implementació).

Interfície Implementador

// Implementador
public interface Color {
    void applyColor();
}

Concret Implementador

// Concret Implementador
public class RedColor implements Color {
    @Override
    public void applyColor() {
        System.out.println("Applying red color");
    }
}

public class BlueColor implements Color {
    @Override
    public void applyColor() {
        System.out.println("Applying blue color");
    }
}

Abstracció

// Abstracció
public abstract class Shape {
    protected Color color;

    protected Shape(Color color) {
        this.color = color;
    }

    abstract void draw();
}

Refinament de l'Abstracció

// Refinament de l'Abstracció
public class Circle extends Shape {
    public Circle(Color color) {
        super(color);
    }

    @Override
    void draw() {
        System.out.print("Drawing Circle with color: ");
        color.applyColor();
    }
}

public class Square extends Shape {
    public Square(Color color) {
        super(color);
    }

    @Override
    void draw() {
        System.out.print("Drawing Square with color: ");
        color.applyColor();
    }
}

Client

public class Client {
    public static void main(String[] args) {
        Shape redCircle = new Circle(new RedColor());
        Shape blueSquare = new Square(new BlueColor());

        redCircle.draw();
        blueSquare.draw();
    }
}

Avantatges del Patró Bridge

  • Separació de preocupacions: L'abstracció i la implementació es poden desenvolupar i evolucionar independentment.
  • Reducció de la complexitat: Evita la combinació de múltiples dimensions de variació en una sola jerarquia de classes.
  • Flexibilitat: Facilita l'extensió de les classes sense afectar les altres.

Desavantatges del Patró Bridge

  • Complexitat inicial: Pot afegir complexitat inicial al disseny del sistema.
  • Més classes: Pot requerir la creació de més classes i interfícies.

Conclusió

El patró Bridge és una solució elegant per separar l'abstracció de la seva implementació, permetent que ambdues puguin variar independentment. Això és especialment útil en sistemes complexos on es vol evitar una explosió de classes derivades. Tot i que pot afegir complexitat inicial, els beneficis a llarg termini en termes de mantenibilitat i extensibilitat sovint superen els inconvenients.
