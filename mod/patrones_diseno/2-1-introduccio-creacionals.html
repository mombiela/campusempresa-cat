Introducció als Patrons Creacionals

Els patrons creacionals són un tipus de patrons de disseny que tracten amb la creació d'objectes. Aquests patrons proporcionen mecanismes per crear objectes de manera que augmenten la flexibilitat i la reutilització del codi. En lloc de crear objectes directament amb el constructor, els patrons creacionals utilitzen diferents tècniques per controlar aquest procés.

Objectius dels Patrons Creacionals

  1. Encapsulació del procés de creació: Permeten encapsular la lògica de creació d'objectes, separant-la de la lògica del negoci.
  2. Reducció de la complexitat: Simplifiquen la creació d'objectes complexos.
  3. Flexibilitat i reutilització: Faciliten la reutilització del codi i permeten canviar la manera com es creen els objectes sense afectar altres parts del sistema.

Tipus de Patrons Creacionals

Els principals patrons creacionals són:

  1. Singleton: Garanteix que una classe només tingui una instància i proporciona un punt d'accés global a aquesta instància.
  2. Factory Method: Defineix una interfície per crear un objecte, però permet que les subclasses decideixin quin tipus d'objecte crear.
  3. Abstract Factory: Proporciona una interfície per crear famílies d'objectes relacionats o dependents sense especificar les seves classes concretes.
  4. Builder: Separa la construcció d'un objecte complex de la seva representació, de manera que el mateix procés de construcció pot crear diferents representacions.
  5. Prototype: Permet crear nous objectes copiant una instància existent, utilitzant un prototip com a base.

Comparació dels Patrons Creacionals

Patró Propòsit Quan Utilitzar-lo
Singleton Assegurar que una classe només tingui una instància. Quan es necessita un únic punt d'accés global a una instància.
Factory Method Definir una interfície per crear objectes, permetent a les subclasses decidir la seva creació. Quan les subclasses necessiten especificar els objectes que han de ser creats.
Abstract Factory Proporcionar una interfície per crear famílies d'objectes relacionats. Quan es necessita crear famílies d'objectes relacionats o dependents.
Builder Separar la construcció d'un objecte complex de la seva representació. Quan es necessita crear diferents representacions d'un objecte complex.
Prototype Crear nous objectes copiant un prototip existent. Quan la creació d'objectes és costosa i es poden reutilitzar instàncies existents.

Exemples de Codi

Singleton

public class Singleton {
    private static Singleton instance;

    private Singleton() {
        // Constructor privat per evitar la instanciació directa
    }

    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

Factory Method

abstract class Product {
    abstract void use();
}

class ConcreteProductA extends Product {
    void use() {
        System.out.println("Using Product A");
    }
}

class ConcreteProductB extends Product {
    void use() {
        System.out.println("Using Product B");
    }
}

abstract class Creator {
    abstract Product factoryMethod();

    void anOperation() {
        Product product = factoryMethod();
        product.use();
    }
}

class ConcreteCreatorA extends Creator {
    Product factoryMethod() {
        return new ConcreteProductA();
    }
}

class ConcreteCreatorB extends Creator {
    Product factoryMethod() {
        return new ConcreteProductB();
    }
}

Conclusió

Els patrons creacionals són fonamentals per a la creació d'objectes en el desenvolupament de programari. Proporcionen solucions estructurades i flexibles que permeten gestionar la complexitat de la creació d'objectes, millorant la reutilització del codi i la seva mantenibilitat. En els següents temes, explorarem cada un d'aquests patrons en detall, amb exemples pràctics i exercicis per consolidar el coneixement.
