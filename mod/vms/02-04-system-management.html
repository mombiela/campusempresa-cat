Comandes de Gestió del Sistema
Següent ►

En aquest tema, aprendrem sobre les comandes de gestió del sistema a OpenVMS. Aquestes comandes són essencials per a l'administració i manteniment del sistema, permetent als administradors gestionar recursos, supervisar el rendiment i assegurar la seguretat del sistema.

Objectius del Tema

  • Entendre les comandes bàsiques de gestió del sistema a OpenVMS.
  • Aprendre a utilitzar aquestes comandes per a tasques administratives comunes.
  • Familiaritzar-se amb les opcions i paràmetres més importants de cada comanda.

  1. Comandes de Gestió del Sistema

1.1. Comanda SHOW SYSTEM

La comanda SHOW SYSTEM proporciona una visió general de l'estat actual del sistema, incloent informació sobre processos actius, ús de la CPU, memòria i altres recursos.

Sintaxi

SHOW SYSTEM

Exemple

$ SHOW SYSTEM

Explicació

Aquesta comanda mostra una llista de tots els processos actius al sistema, juntament amb informació com el nom del procés, l'usuari, l'ús de la CPU i l'estat del procés.

1.2. Comanda MONITOR

La comanda MONITOR s'utilitza per supervisar el rendiment del sistema en temps real. Proporciona informació detallada sobre l'ús de la CPU, memòria, discos i altres recursos.

Sintaxi

MONITOR [opcions]

Opcions Comunes

  • MONITOR SYSTEM: Mostra una visió general del rendiment del sistema.
  • MONITOR DISK: Mostra l'activitat dels discos.
  • MONITOR PROCESS: Mostra informació detallada sobre els processos.

Exemple

$ MONITOR SYSTEM

Explicació

Aquesta comanda mostra una visió general del rendiment del sistema, incloent l'ús de la CPU, memòria i altres recursos en temps real.

1.3. Comanda SET

La comanda SET s'utilitza per configurar diversos paràmetres del sistema, com ara les variables d'entorn, les propietats dels dispositius i altres configuracions del sistema.

Sintaxi

SET [opcions]

Opcions Comunes

  • SET DEFAULT [directori]: Canvia el directori de treball per defecte.
  • SET PROCESS/PRIORITY=[prioritat]: Canvia la prioritat d'un procés.
  • SET DEVICE [dispositiu] [paràmetres]: Configura les propietats d'un dispositiu.

Exemple

$ SET DEFAULT SYS$SYSROOT:[SYSMGR]

Explicació

Aquesta comanda canvia el directori de treball per defecte al directori de gestió del sistema.

1.4. Comanda START/QUEUE

La comanda START/QUEUE s'utilitza per iniciar una cua de treball, com ara una cua d'impressió o una cua de processos per lots.

Sintaxi

START/QUEUE [nom_de_la_cua]

Exemple

$ START/QUEUE SYS$PRINT

Explicació

Aquesta comanda inicia la cua d'impressió anomenada SYS$PRINT, permetent que els treballs d'impressió s'enviïn a la impressora associada.

1.5. Comanda STOP/QUEUE

La comanda STOP/QUEUE s'utilitza per aturar una cua de treball.

Sintaxi

STOP/QUEUE [nom_de_la_cua]

Exemple

$ STOP/QUEUE SYS$PRINT

Explicació

Aquesta comanda atura la cua d'impressió anomenada SYS$PRINT, impedint que nous treballs d'impressió s'enviïn a la impressora associada.

  1. Exercicis Pràctics

Exercici 1: Supervisar el Sistema

Utilitza la comanda SHOW SYSTEM per obtenir una llista de tots els processos actius al sistema. Identifica el procés amb el major ús de la CPU.

Solució

$ SHOW SYSTEM

Busca el procés amb el major valor a la columna d'ús de la CPU.

Exercici 2: Canviar el Directori de Treball

Utilitza la comanda SET DEFAULT per canviar el directori de treball per defecte al directori SYS$SYSROOT:[SYSMGR].

Solució

$ SET DEFAULT SYS$SYSROOT:[SYSMGR]

Exercici 3: Iniciar i Aturar una Cua d'Impressió

Inicia la cua d'impressió SYS$PRINT i després atura-la.

Solució

$ START/QUEUE SYS$PRINT
$ STOP/QUEUE SYS$PRINT

Resum

En aquest tema, hem après sobre les comandes de gestió del sistema a OpenVMS, incloent SHOW SYSTEM, MONITOR, SET, START/QUEUE i STOP/QUEUE. Aquestes comandes són essencials per a l'administració i manteniment del sistema, permetent als administradors gestionar recursos, supervisar el rendiment i assegurar la seguretat del sistema. Hem practicat l'ús d'aquestes comandes amb exercicis pràctics per reforçar els conceptes apresos.
