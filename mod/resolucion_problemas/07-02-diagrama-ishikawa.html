Diagrama d'Ishikawa

Següent ►

Introducció

El Diagrama d'Ishikawa, també conegut com a Diagrama de Causa-Efecte o Diagrama de Peix, és una eina visual utilitzada per identificar, explorar i representar gràficament les possibles causes d'un problema específic. Aquesta tècnica va ser desenvolupada pel professor Kaoru Ishikawa als anys 60 i és àmpliament utilitzada en la gestió de qualitat i la resolució de problemes.

Objectius del Diagrama d'Ishikawa

  • Identificar les causes principals i secundàries d'un problema.
  • Facilitar la comprensió de les relacions entre diferents factors que contribueixen a un problema.
  • Proporcionar una visió clara i estructurada per a l'anàlisi de problemes complexos.
  • Ajudar en la generació d'idees per a la resolució de problemes.

Components del Diagrama d'Ishikawa

  1. Cap del Peix (Efecte): Representa el problema o efecte que es vol analitzar.
  2. Espina dorsal: Una línia horitzontal que connecta el cap del peix amb les diferents causes.
  3. Espines principals: Línies diagonals que surten de l'espina dorsal i representen les categories principals de causes.
  4. Espines secundàries: Línies que surten de les espines principals i representen les causes específiques dins de cada categoria.

Categories Principals de Causes

Les categories principals de causes poden variar segons el context, però les més comunes són:

  • Mètodes: Procediments, processos i pràctiques utilitzades.
  • Maquinària: Eines, equips i tecnologia.
  • Materials: Matèries primeres, components i subministraments.
  • Mano d'obra: Personal, habilitats i formació.
  • Medi ambient: Condicions ambientals, ubicació i factors externs.
  • Mesures: Mètodes de mesurament, dades i informació.

Passos per Crear un Diagrama d'Ishikawa

  1. Definir el problema: Escriu el problema o efecte a la cap del peix.
  2. Identificar les categories principals: Dibuixa les espines principals i etiqueta-les amb les categories de causes.
  3. Generar causes específiques: Per cada categoria principal, identifica les causes específiques i dibuixa les espines secundàries.
  4. Analitzar el diagrama: Revisa el diagrama per identificar les causes més significatives i prioritza-les per a la seva resolució.

Exemple de Diagrama d'Ishikawa

Problema: Alta taxa de defectes en la producció

Categoria Causes
Mètodes Procediments inadequats, manca d'estandardització
Maquinària Manteniment insuficient, equips obsolets
Materials Qualitat inconsistent, proveïdors no fiables
Mano d'obra Formació insuficient, manca de motivació
Medi ambient Condicions de treball pobres, temperatura inestable
Mesures Dades inexactes, manca de seguiment

Exercici Pràctic

Instruccions

  1. Defineix un problema: Tria un problema real o hipotètic que vulguis analitzar.
  2. Crea el diagrama: Dibuixa un diagrama d'Ishikawa seguint els passos descrits anteriorment.
  3. Identifica les causes: Omple el diagrama amb les causes principals i secundàries.
  4. Analitza i prioritza: Revisa el diagrama i identifica les causes més significatives per abordar.

Solució Exemple

Problema: Retards en el lliurament de projectes

Categoria Causes
Mètodes Planificació inadequada, manca de seguiment
Maquinària Programari obsolet, manca d'eines adequades
Materials Recursos insuficients, manca de subministraments
Mano d'obra Sobrecàrrega de treball, manca de formació
Medi ambient Entorn de treball sorollós, interrupcions freqüents
Mesures Manca de mètriques clares, informes inexactes

Conclusió

El Diagrama d'Ishikawa és una eina poderosa per a la resolució de problemes, ja que permet identificar i analitzar les causes d'un problema de manera estructurada i visual. Utilitzar aquesta tècnica pot ajudar els professionals a abordar problemes complexos de manera més eficient i efectiva, millorant així la qualitat i el rendiment en les seves organitzacions.
