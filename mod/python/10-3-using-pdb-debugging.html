10-3: Ús de pdb per a la Depuració

Introducció

El depurador pdb (Python Debugger) és una eina poderosa que permet als desenvolupadors inspeccionar i controlar l'execució del codi Python per identificar i corregir errors. En aquesta secció, aprendrem com utilitzar pdb per depurar els nostres programes.

Conceptes Clau

  • Depuració: El procés de trobar i corregir errors en el codi.
  • Breakpoint: Un punt en el codi on l'execució es deté per permetre la inspecció.
  • Comandes de pdb: Ordres específiques que es poden utilitzar per controlar l'execució del programa durant la depuració.

Configuració Inicial

Per començar a utilitzar pdb, primer hem d'importar el mòdul pdb en el nostre codi. Aquí teniu un exemple bàsic:

import pdb

def suma(a, b):
    resultat = a + b
    return resultat

pdb.set_trace()  # Estableix un breakpoint aquí
print(suma(3, 4))

Comandes Bàsiques de pdb

Un cop el programa es deté en un breakpoint, podem utilitzar diverses comandes per inspeccionar i controlar l'execució del codi:

Comanda Descripció
n Executa la següent línia de codi (next).
c Continua l'execució fins al següent breakpoint (continue).
l Mostra el codi al voltant de la línia actual (list).
p Imprimeix el valor d'una variable (print).
q Surt del depurador (quit).

Exemple Pràctic

Vegem un exemple més complex per entendre millor com utilitzar pdb:

import pdb

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n - 1)

def main():
    pdb.set_trace()  # Estableix un breakpoint aquí
    num = 5
    resultat = factorial(num)
    print(f"El factorial de {num} és {resultat}")

if __name__ == "__main__":
    main()

Explicació del Codi

  1. Importació de pdb: Importem el mòdul pdb.
  2. Funció factorial: Definim una funció recursiva per calcular el factorial d'un nombre.
  3. Funció main: Establim un breakpoint amb pdb.set_trace() abans de calcular el factorial.
  4. Execució del programa: Quan executem el programa, s'aturarà en el breakpoint, permetent-nos inspeccionar les variables i controlar l'execució.

Exercici Pràctic

Exercici: Depura el següent codi per trobar i corregir l'error:

import pdb

def divisio(a, b):
    pdb.set_trace()  # Estableix un breakpoint aquí
    return a / b

resultat = divisio(10, 0)
print(f"El resultat és {resultat}")

Solució:

  1. Executa el codi.
  2. Utilitza pdb per inspeccionar les variables a i b.
  3. Identifica que b és 0, el que causa una divisió per zero.
  4. Corregeix l'error afegint una comprovació abans de la divisió:
import pdb

def divisio(a, b):
    pdb.set_trace()  # Estableix un breakpoint aquí
    if b == 0:
        return "Error: Divisió per zero"
    return a / b

resultat = divisio(10, 0)
print(f"El resultat és {resultat}")

Errors Comuns i Consells

  • Oblidar establir breakpoints: Assegura't d'establir breakpoints en els punts crítics del codi.
  • No inspeccionar les variables correctament: Utilitza la comanda p per imprimir i inspeccionar les variables.
  • Saltar-se línies importants: Utilitza n amb cura per no saltar-te línies importants del codi.

Conclusió

El depurador pdb és una eina essencial per a qualsevol desenvolupador Python. Amb les comandes bàsiques i una mica de pràctica, pots identificar i corregir errors de manera eficient. Practica amb diferents exemples per familiaritzar-te amb pdb i millorar les teves habilitats de depuració.
