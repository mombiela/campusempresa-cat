6-3: Gestió d'Excepcions

Introducció

La gestió d'excepcions és una part fonamental de la programació en Python. Les excepcions són errors que es produeixen durant l'execució d'un programa. En lloc de permetre que el programa falli, Python ofereix mecanismes per capturar i gestionar aquestes excepcions, permetent que el programa continuï executant-se o finalitzi de manera controlada.

Conceptes Clau

  1. Excepcions: Errors que es produeixen durant l'execució d'un programa.
  2. Blocs try i except: Estructures per capturar i gestionar excepcions.
  3. Blocs else i finally: Estructures opcionals per afegir lògica addicional després de gestionar excepcions.
  4. Llançament d'Excepcions: Crear i llançar les teves pròpies excepcions.

Blocs `try` i `except`

Els blocs try i except són la base de la gestió d'excepcions en Python. El codi que pot produir una excepció es col·loca dins del bloc try, i el codi per gestionar l'excepció es col·loca dins del bloc except.

try:
    # Codi que pot produir una excepció
    resultat = 10 / 0
except ZeroDivisionError:
    # Codi per gestionar l'excepció
    print("Error: No es pot dividir per zero.")

Blocs `else` i `finally`

Els blocs else i finally són opcionals però poden ser molt útils. El bloc else s'executa si no es produeix cap excepció, i el bloc finally s'executa sempre, independentment de si es produeix una excepció o no.

try:
    resultat = 10 / 2
except ZeroDivisionError:
    print("Error: No es pot dividir per zero.")
else:
    print("El resultat és:", resultat)
finally:
    print("Aquest bloc s'executa sempre.")

Llançament d'Excepcions

També pots llançar les teves pròpies excepcions utilitzant la paraula clau raise.

def divideix(x, y):
    if y == 0:
        raise ValueError("El divisor no pot ser zero.")
    return x / y

try:
    resultat = divideix(10, 0)
except ValueError as e:
    print("Error:", e)

Exemples Pràctics

Exemple 1: Gestió de Fitxers amb Excepcions

try:
    fitxer = open('fitxer_inexistent.txt', 'r')
    contingut = fitxer.read()
except FileNotFoundError:
    print("Error: El fitxer no existeix.")
finally:
    fitxer.close()

Exemple 2: Entrada d'Usuari amb Validació

try:
    edat = int(input("Introdueix la teva edat: "))
except ValueError:
    print("Error: Has d'introduir un número sencer.")
else:
    print("La teva edat és:", edat)

Exercicis Pràctics

Exercici 1: Divisió amb Gestió d'Excepcions

Escriu una funció que demani a l'usuari dos números i mostri el resultat de la seva divisió. Gestiona l'excepció si l'usuari introdueix un divisor zero.

def divisio_amb_excepcions():
    try:
        num1 = float(input("Introdueix el primer número: "))
        num2 = float(input("Introdueix el segon número: "))
        resultat = num1 / num2
    except ZeroDivisionError:
        print("Error: No es pot dividir per zero.")
    except ValueError:
        print("Error: Has d'introduir un número vàlid.")
    else:
        print("El resultat de la divisió és:", resultat)

divisio_amb_excepcions()

Exercici 2: Lectura de Fitxers amb Gestió d'Excepcions

Escriu un programa que llegeixi un fitxer de text i mostri el seu contingut. Gestiona l'excepció si el fitxer no existeix.

def lectura_fitxer():
    try:
        nom_fitxer = input("Introdueix el nom del fitxer: ")
        with open(nom_fitxer, 'r') as fitxer:
            contingut = fitxer.read()
            print(contingut)
    except FileNotFoundError:
        print("Error: El fitxer no existeix.")
    except Exception as e:
        print("S'ha produït un error:", e)

lectura_fitxer()

Resum

En aquesta secció, hem après com gestionar excepcions en Python utilitzant els blocs try, except, else i finally. També hem vist com llançar les nostres pròpies excepcions amb raise. La gestió d'excepcions és crucial per escriure codi robust i fiable que pugui gestionar errors de manera controlada.
