11-1: Empaquetat i Distribució d'Aplicacions

Introducció

En aquesta secció, aprendrem com empaquetar i distribuir aplicacions Python. Això és essencial per compartir el teu treball amb altres desenvolupadors o per desplegar aplicacions en entorns de producció. Cobriràs conceptes com la creació de paquets, la gestió de dependències i la publicació de paquets a PyPI (Python Package Index).

Objectius

  • Entendre la importància de l'empaquetat i la distribució.
  • Aprendre a crear un paquet Python.
  • Gestionar les dependències del projecte.
  • Publicar un paquet a PyPI.

Conceptes Clau

  1. Empaquetat: El procés de preparar el teu codi perquè pugui ser fàcilment distribuït i instal·lat.
  2. Distribució: La publicació del teu paquet perquè altres puguin utilitzar-lo.
  3. PyPI: El repositori oficial de paquets Python.

Creació d'un Paquet Python

Estructura del Projecte

Una estructura típica per a un projecte Python empaquetat podria ser:

my_package/
├── my_package/
│   ├── __init__.py
│   ├── module1.py
│   └── module2.py
├── tests/
│   ├── __init__.py
│   └── test_module1.py
├── setup.py
└── README.md
  • my_package/: El directori principal del teu paquet.
  • __init__.py: Fa que el directori sigui tractat com un paquet Python.
  • setup.py: El fitxer de configuració per a l'empaquetat.
  • README.md: Documentació del paquet.

Fitxer setup.py

El fitxer setup.py és crucial per a l'empaquetat. Aquí tens un exemple bàsic:

from setuptools import setup, find_packages

setup(
    name='my_package',
    version='0.1',
    packages=find_packages(),
    install_requires=[
        'requests',
        'numpy',
    ],
    author='El teu Nom',
    author_email='el.teu.email@example.com',
    description='Una breu descripció del paquet',
    long_description=open('README.md').read(),
    long_description_content_type='text/markdown',
    url='https://github.com/el-teu-usuari/my_package',
    classifiers=[
        'Programming Language :: Python :: 3',
        'License :: OSI Approved :: MIT License',
        'Operating System :: OS Independent',
    ],
    python_requires='>=3.6',
)

Gestió de Dependències

Les dependències es defineixen en el camp install_requires del fitxer setup.py. Això assegura que les biblioteques necessàries s'instal·lin automàticament quan algú instal·li el teu paquet.

Publicació a PyPI

Creació d'un Compte a PyPI

  1. Registra't a PyPI.
  2. Verifica el teu compte a través del correu electrònic.

Construcció del Paquet

Utilitza setuptools i wheel per construir el paquet:

python setup.py sdist bdist_wheel

Això crearà un directori dist/ amb els fitxers del paquet.

Publicació del Paquet

Utilitza twine per pujar el paquet a PyPI:

pip install twine
twine upload dist/*

Se't demanarà el teu nom d'usuari i contrasenya de PyPI.

Objectiu

Crea i publica un paquet Python senzill que contingui una funció que sumi dos nombres.

Passos

  1. Crea l'estructura del projecte:
sum_package/
├── sum_package/
│   ├── __init__.py
│   └── sum.py
├── setup.py
└── README.md
  1. Implementa la funció en sum.py:
def add(a, b):
    return a + b
  1. Escriu el fitxer setup.py:
from setuptools import setup, find_packages

setup(
    name='sum_package',
    version='0.1',

    long_description=open('README.md').read(),
    long_description_content_type='text/markdown',

        'Programming Language :: Python :: 3',

        'Operating System :: OS Independent',
    ],
    python_requires='>=3.6',
)
