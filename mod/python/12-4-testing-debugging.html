12-4: Proves i Depuració

En aquesta secció, ens centrarem en la importància de les proves i la depuració en el desenvolupament de programari. Aprendrem com assegurar-nos que el nostre projecte final funcioni correctament i com identificar i corregir errors.

Objectius d'Aprenentatge

  • Comprendre la importància de les proves en el desenvolupament de programari.
  • Aprendre a escriure proves unitàries per al projecte.
  • Utilitzar eines de depuració per identificar i corregir errors.

Contingut

1. Importància de les Proves

Les proves són essencials per assegurar que el codi funcioni com s'espera. Ajuden a detectar errors abans que el programari arribi als usuaris finals.

Tipus de Proves:

  • Proves Unitàries: Verifiquen el funcionament de components individuals del codi.
  • Proves d'Integració: Asseguren que diferents components funcionin bé junts.
  • Proves de Sistema: Verifiquen el sistema complet en un entorn que simula l'ús real.

2. Escriptura de Proves Unitàries

Les proves unitàries són una part fonamental del desenvolupament de programari. En Python, podem utilitzar el mòdul unittest per escriure aquestes proves.

Exemple de Prova Unitària:

import unittest

def suma(a, b):
    return a + b

class TestSuma(unittest.TestCase):
    def test_suma_positius(self):
        self.assertEqual(suma(1, 2), 3)

    def test_suma_negatius(self):
        self.assertEqual(suma(-1, -1), -2)

if __name__ == '__main__':
    unittest.main()

Explicació:

  • unittest.TestCase: Classe base per a totes les proves unitàries.
  • self.assertEqual: Comprova que el resultat de suma(1, 2) sigui 3.
  • if __name__ == '__main__': Executa les proves quan el fitxer es corre directament.

3. Tècniques de Depuració

La depuració és el procés d'identificar i corregir errors en el codi. Python proporciona diverses eines per a la depuració.

Ús de print Statements: Una tècnica simple però efectiva és utilitzar print statements per veure el valor de les variables en diferents punts del codi.

Ús de pdb: El mòdul pdb és un depurador interactiu per a Python.

Exemple d'Ús de pdb:

import pdb

def suma(a, b):
    pdb.set_trace()
    return a + b

resultat = suma(1, 2)
print(resultat)

Comandes bàsiques de pdb:

  • n: Executa la següent línia de codi.
  • c: Continua l'execució fins al següent punt de trencament.
  • q: Surt del depurador.

4. Exercicis Pràctics

Exercici 1: Escriure Proves Unitàries Escriu proves unitàries per a la següent funció que calcula el factorial d'un nombre:

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

Solució:

import unittest

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

class TestFactorial(unittest.TestCase):
    def test_factorial_zero(self):
        self.assertEqual(factorial(0), 1)

    def test_factorial_positiu(self):
        self.assertEqual(factorial(5), 120)

if __name__ == '__main__':
    unittest.main()

Exercici 2: Depurar un Error El següent codi conté un error. Utilitza pdb per identificar i corregir l'error.

def divideix(a, b):
    return a / b

resultat = divideix(4, 0)
print(resultat)

Solució:

import pdb

def divideix(a, b):
    pdb.set_trace()
    if b == 0:
        return "Error: Divisió per zero"
    return a / b

resultat = divideix(4, 0)
print(resultat)

Resum

En aquesta secció, hem après la importància de les proves i la depuració en el desenvolupament de programari. Hem vist com escriure proves unitàries utilitzant unittest i com utilitzar pdb per depurar el codi. Aquestes habilitats són essencials per assegurar que el nostre projecte final funcioni correctament i estigui lliure d'errors.

En la següent secció, ens centrarem en el desplegament del projecte final.
