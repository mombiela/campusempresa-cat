12-5: Desplegament

Introducció

El desplegament és l'última fase del cicle de vida del desenvolupament de programari, on l'aplicació es posa en producció perquè els usuaris finals puguin accedir-hi. En aquesta secció, aprendrem com desplegar una aplicació web desenvolupada amb Flask a un servidor de producció.

Objectius

  • Entendre els conceptes bàsics del desplegament.
  • Configurar un servidor de producció per a una aplicació Flask.
  • Utilitzar eines com Gunicorn i Nginx per gestionar el desplegament.
  • Desplegar l'aplicació a una plataforma de núvol com Heroku.

Conceptes Bàsics del Desplegament

Abans de començar amb el desplegament pràctic, és important entendre alguns conceptes bàsics:

  • Servidor de Producció: Un servidor optimitzat per executar aplicacions en un entorn de producció, amb alta disponibilitat i seguretat.
  • WSGI (Web Server Gateway Interface): Un estàndard per a servidors web i aplicacions web en Python que facilita la comunicació entre ells.
  • Gunicorn: Un servidor WSGI per a aplicacions Python que és fàcil d'utilitzar i molt eficient.
  • Nginx: Un servidor web que pot actuar com a proxy invers per a la vostra aplicació, millorant la seguretat i el rendiment.

Configuració del Servidor de Producció

1. Preparar l'Entorn

Abans de desplegar l'aplicació, assegura't que tens un servidor de producció configurat. Pots utilitzar un servei de núvol com DigitalOcean, AWS, o qualsevol altre proveïdor de VPS.

2. Instal·lar les Dependències

Instal·la les dependències necessàries al servidor:

sudo apt update
sudo apt install python3-pip python3-dev nginx
pip3 install virtualenv

3. Configurar l'Entorn Virtual

Crea i activa un entorn virtual:

mkdir ~/myproject
cd ~/myproject
virtualenv myprojectenv
source myprojectenv/bin/activate

4. Instal·lar les Dependències de l'Aplicació

Instal·la les dependències de l'aplicació dins de l'entorn virtual:

pip install flask gunicorn

5. Crear el Fitxer WSGI

Crea un fitxer wsgi.py per a la teva aplicació:

from myproject import app

if __name__ == "__main__":
    app.run()

6. Configurar Gunicorn

Executa l'aplicació amb Gunicorn:

gunicorn --workers 3 wsgi:app

Configuració de Nginx

1. Configurar Nginx com a Proxy Invers

Crea un fitxer de configuració per a Nginx:

sudo nano /etc/nginx/sites-available/myproject

Afegeix la següent configuració:

server {
    listen 80;
    server_name your_domain_or_IP;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Enllaça el fitxer de configuració i reinicia Nginx:

sudo ln -s /etc/nginx/sites-available/myproject /etc/nginx/sites-enabled
sudo nginx -t
sudo systemctl restart nginx

Desplegament a Heroku

1. Preparar l'Aplicació

Crea un fitxer Procfile a l'arrel del teu projecte:

web: gunicorn wsgi:app

2. Crear un Entorn Virtual i Instal·lar les Dependències

pip freeze > requirements.txt

3. Desplegar l'Aplicació

Inicia sessió a Heroku i crea una nova aplicació:

heroku login
heroku create myproject

Puja el codi al repositori de Heroku:

git init
heroku git:remote -a myproject
git add .
git commit -m "Initial commit"
git push heroku master

Exercici Pràctic

Desplega una aplicació Flask simple a Heroku seguint els passos anteriors. Assegura't que l'aplicació funcioni correctament en producció.

Solució de l'Exercici

  1. Crea l'aplicació Flask:
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello_world():
    return 'Hello, World!'

if __name__ == '__main__':
    app.run()
  1. Crea el fitxer Procfile:
web: gunicorn wsgi:app
  1. Crea el fitxer requirements.txt:
Flask==2.0.1
gunicorn==20.1.0
  1. Desplega l'aplicació a Heroku:
heroku login
heroku create myflaskapp
git init
heroku git:remote -a myflaskapp
git add .
git commit -m "Initial commit"
git push heroku master

Conclusió

En aquesta secció, hem après com desplegar una aplicació Flask a un servidor de producció utilitzant Gunicorn i Nginx, així com a una plataforma de núvol com Heroku. El desplegament és una habilitat essencial per a qualsevol desenvolupador, ja que permet posar les aplicacions a disposició dels usuaris finals. Amb aquests coneixements, estàs preparat per desplegar les teves pròpies aplicacions en entorns de producció.
