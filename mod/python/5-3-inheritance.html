5-3: Herència

Introducció a l'Herència

L'herència és un concepte fonamental en la programació orientada a objectes (OOP) que permet crear una nova classe (subclasse) a partir d'una classe existent (superclasse). La subclasse hereta els atributs i mètodes de la superclasse, cosa que permet reutilitzar codi i afegir noves funcionalitats o modificar les existents.

Conceptes Clau

  • Superclasse (Classe Pare): La classe de la qual s'hereten atributs i mètodes.
  • Subclasse (Classe Filla): La classe que hereta atributs i mètodes de la superclasse.
  • Herència Simple: Una subclasse hereta d'una sola superclasse.
  • Herència Múltiple: Una subclasse hereta de més d'una superclasse (no es recomana per la seva complexitat).

Sintaxi Bàsica

class SuperClasse:
    def __init__(self, atribut):
        self.atribut = atribut

    def metode(self):
        print("Aquest és un mètode de la superclasse")

class SubClasse(SuperClasse):
    def __init__(self, atribut, nou_atribut):
        super().__init__(atribut)
        self.nou_atribut = nou_atribut

    def nou_metode(self):
        print("Aquest és un mètode de la subclasse")

Exemple Pràctic

Superclasse: Animal

class Animal:
    def __init__(self, nom):
        self.nom = nom

    def parlar(self):
        raise NotImplementedError("Aquest mètode ha de ser implementat per subclasses")

    def moure(self):
        print(f"{self.nom} s'està movent")

Subclasse: Gos

class Gos(Animal):
    def parlar(self):
        print(f"{self.nom} diu: Guau!")

Subclasse: Gat

class Gat(Animal):
    def parlar(self):
        print(f"{self.nom} diu: Miau!")

Ús de les Classes

gos = Gos("Rex")
gos.moure()  # Rex s'està movent
gos.parlar()  # Rex diu: Guau!

gat = Gat("Whiskers")
gat.moure()  # Whiskers s'està movent
gat.parlar()  # Whiskers diu: Miau!

Exercicis Pràctics

Exercici 1: Creació de Subclasses

Descripció: Crea una superclasse Vehicle amb un mètode moure. Després, crea dues subclasses Cotxe i Bicicleta que heretin de Vehicle i implementin el mètode moure de manera diferent.

Solució:

class Vehicle:
    def __init__(self, nom):
        self.nom = nom

    def moure(self):
        raise NotImplementedError("Aquest mètode ha de ser implementat per subclasses")

class Cotxe(Vehicle):
    def moure(self):
        print(f"{self.nom} està conduint")

class Bicicleta(Vehicle):
    def moure(self):
        print(f"{self.nom} està pedalant")

cotxe = Cotxe("Toyota")
cotxe.moure()  # Toyota està conduint

bicicleta = Bicicleta("Mountain Bike")
bicicleta.moure()  # Mountain Bike està pedalant

Exercici 2: Atributs Addicionals en Subclasses

Descripció: Crea una superclasse Persona amb atributs nom i edat. Després, crea una subclasse Estudiant que afegeixi l'atribut curs.

Solució:

class Persona:
    def __init__(self, nom, edat):
        self.nom = nom
        self.edat = edat

class Estudiant(Persona):
    def __init__(self, nom, edat, curs):
        super().__init__(nom, edat)
        self.curs = curs

    def info(self):
        print(f"Nom: {self.nom}, Edat: {self.edat}, Curs: {self.curs}")

estudiant = Estudiant("Anna", 20, "Matemàtiques")
estudiant.info()  # Nom: Anna, Edat: 20, Curs: Matemàtiques

Errors Comuns i Consells

  • No Implementar Mètodes Abstractes: Si una superclasse defineix un mètode que ha de ser implementat per les subclasses, assegura't d'implementar-lo.
  • Ús Incorrecte de super(): Utilitza super() per cridar el constructor de la superclasse i assegurar-te que els atributs de la superclasse es configuren correctament.
  • Herència Múltiple: Evita l'herència múltiple si és possible, ja que pot complicar el codi i fer-lo més difícil de mantenir.

Conclusió

L'herència és una eina poderosa en la programació orientada a objectes que permet reutilitzar codi i crear jerarquies de classes. Comprendre com utilitzar l'herència de manera efectiva és crucial per escriure codi net i mantenible. En el següent tema, explorarem el polimorfisme, que permet utilitzar objectes de diferents classes de manera intercanviable.
