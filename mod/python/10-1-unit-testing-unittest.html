10-1: Proves Unitàries amb unittest

Introducció a les Proves Unitàries

Les proves unitàries són una tècnica de verificació de programari en què es comprova el funcionament correcte de les unitats més petites del codi, com ara funcions o mètodes. En Python, el mòdul unittest proporciona una infraestructura per crear i executar proves unitàries.

Objectius d'Aquesta Secció

  • Comprendre la importància de les proves unitàries.
  • Aprendre a utilitzar el mòdul unittest per escriure i executar proves.
  • Conèixer les bones pràctiques en l'escriptura de proves unitàries.

Conceptes Clau

  • Test Case: Una unitat de prova que comprova una funcionalitat específica.
  • Test Suite: Un conjunt de test cases.
  • Test Runner: Una eina que executa test cases i reporta els resultats.

Configuració de l'Entorn

Abans de començar, assegura't de tenir Python instal·lat al teu sistema. El mòdul unittest ve inclòs amb la instal·lació estàndard de Python, així que no cal instal·lar res addicional.

Estructura Bàsica d'una Prova amb unittest

A continuació es mostra un exemple bàsic d'una prova unitària utilitzant unittest.

import unittest

def suma(a, b):
    return a + b

class TestSuma(unittest.TestCase):
    def test_suma_positius(self):
        self.assertEqual(suma(1, 2), 3)

    def test_suma_negatius(self):
        self.assertEqual(suma(-1, -1), -2)

    def test_suma_zero(self):
        self.assertEqual(suma(0, 0), 0)

if __name__ == '__main__':
    unittest.main()

Explicació del Codi

  1. Importació del mòdul unittest:

    import unittest

  2. Definició de la funció a provar:

    def suma(a, b):
    return a + b

  3. Creació d'una classe de prova:

    class TestSuma(unittest.TestCase):

  4. Definició de test cases:

    • test_suma_positius: Comprova que la suma de dos nombres positius és correcta.
    • test_suma_negatius: Comprova que la suma de dos nombres negatius és correcta.
    • test_suma_zero: Comprova que la suma de dos zeros és correcta.

  5. Execució de les proves:

    if __name__ == '__main__':
    unittest.main()

Bones Pràctiques

  • Nom de les Proves: Utilitza noms descriptius per als teus test cases.
  • Cobertura: Assegura't de cobrir tots els casos possibles, incloent-hi els límits i els casos especials.
  • Aïllament: Cada prova ha de ser independent i no ha de dependre de l'estat deixat per altres proves.

Exercicis Pràctics

Exercici 1: Prova de Funció de Multiplicació

Escriu una funció multiplica(a, b) i crea test cases per comprovar el seu funcionament.

def multiplica(a, b):
    return a * b

class TestMultiplica(unittest.TestCase):
    def test_multiplica_positius(self):
        self.assertEqual(multiplica(2, 3), 6)

    def test_multiplica_negatius(self):
        self.assertEqual(multiplica(-2, -3), 6)

    def test_multiplica_zero(self):
        self.assertEqual(multiplica(0, 5), 0)

if __name__ == '__main__':
    unittest.main()

Exercici 2: Prova de Funció de Divisió

Escriu una funció divideix(a, b) que divideixi a per b i crea test cases per comprovar el seu funcionament, incloent-hi la gestió d'errors per divisió per zero.

def divideix(a, b):
    if b == 0:
        raise ValueError("Divisió per zero")
    return a / b

class TestDivideix(unittest.TestCase):
    def test_divideix_positius(self):
        self.assertEqual(divideix(6, 3), 2)

    def test_divideix_negatius(self):
        self.assertEqual(divideix(-6, -3), 2)

    def test_divideix_zero(self):
        with self.assertRaises(ValueError):
            divideix(5, 0)

if __name__ == '__main__':
    unittest.main()

Errors Comuns i Consells

  • No Aïllar les Proves: Assegura't que cada prova és independent.
  • No Cobrir Tots els Casos: Prova tots els casos possibles, incloent-hi els límits.
  • No Utilitzar Noms Descriptius: Utilitza noms clars i descriptius per als teus test cases.

Conclusió

Les proves unitàries són una part essencial del desenvolupament de programari de qualitat. Utilitzant unittest, pots assegurar-te que les teves funcions funcionen correctament i detectar errors abans que arribin a producció. Practica escrivint proves per a les teves funcions i segueix les bones pràctiques per millorar la qualitat del teu codi.
