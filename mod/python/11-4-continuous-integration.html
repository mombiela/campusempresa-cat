11-4: Integració Contínua

Introducció a la Integració Contínua

La Integració Contínua (CI) és una pràctica de desenvolupament de programari en la qual els desenvolupadors integren el seu codi en un repositori compartit de manera freqüent, preferiblement diverses vegades al dia. Cada integració es verifica mitjançant la construcció automàtica i proves per detectar errors tan aviat com sigui possible.

Objectius de la Integració Contínua

  • Detecció Primerenca d'Errors: Identificar problemes en el codi de manera ràpida i freqüent.
  • Millora de la Qualitat del Codi: Assegurar que el codi compleixi amb els estàndards de qualitat mitjançant proves automàtiques.
  • Reducció del Temps de Lliurament: Facilitar un cicle de desenvolupament més ràpid i eficient.

Components Clau de la Integració Contínua

  1. Repositori de Codi: Un sistema de control de versions (com Git) on es guarda el codi font.
  2. Servidor de CI: Una eina que automatitza la construcció i les proves del codi (com Jenkins, Travis CI, CircleCI).
  3. Scripts de Construcció: Scripts que defineixen com construir i provar el codi.
  4. Proves Automatitzades: Conjunt de proves que s'executen automàticament per verificar la funcionalitat del codi.

Configuració de la Integració Contínua amb GitHub i Travis CI

Pas 1: Crear un Repositori a GitHub

  1. Inicia sessió a GitHub.
  2. Crea un nou repositori per al teu projecte.

Pas 2: Afegir un Fitxer .travis.yml

Crea un fitxer anomenat .travis.yml a l'arrel del teu projecte. Aquest fitxer conté la configuració necessària per a Travis CI.

language: python
python:
  - "3.8"
# Instal·la les dependències
install:
  - pip install -r requirements.txt
# Executa les proves
script:
  - pytest

Pas 3: Connectar Travis CI amb GitHub

  1. Inicia sessió a Travis CI amb el teu compte de GitHub.
  2. Autoritza Travis CI a accedir als teus repositoris.
  3. Activa el repositori que has creat per al teu projecte.

Pas 4: Proves i Construcció Automàtica

Cada vegada que facis un commit al teu repositori de GitHub, Travis CI construirà automàticament el projecte i executarà les proves definides al fitxer .travis.yml.

Exemple Pràctic

Suposem que tens un projecte Python amb la següent estructura:

my_project/
├── my_module.py
├── test_my_module.py
├── requirements.txt
└── .travis.yml

my_module.py

def add(a, b):
    return a + b

test_my_module.py

import pytest
from my_module import add

def test_add():
    assert add(1, 2) == 3
    assert add(-1, 1) == 0
    assert add(0, 0) == 0

requirements.txt

pytest

.travis.yml

language: python
python:
  - "3.8"
install:
  - pip install -r requirements.txt
script:
  - pytest

Exercici Pràctic

  1. Crea un Repositori a GitHub: Crea un nou repositori anomenat ci-example.
  2. Configura Travis CI: Segueix els passos per connectar Travis CI amb el teu repositori.
  3. Afegir Fitxers: Afegeix els fitxers my_module.py, test_my_module.py, requirements.txt i .travis.yml al teu repositori.
  4. Fer un Commit i Push: Fes un commit i push dels fitxers al repositori.
  5. Verifica la Construcció: Verifica que Travis CI executi les proves correctament.

Solució de l'Exercici

  1. Crea un Repositori a GitHub: GitHub
  2. Configura Travis CI: Travis CI
  3. Afegir Fitxers: Utilitza els exemples proporcionats anteriorment.
  4. Fer un Commit i Push:
git add .
git commit -m "Afegir fitxers de projecte"
git push origin main
  1. Verifica la Construcció: Visita Travis CI per veure l'estat de la construcció.

Errors Comuns i Consells

  • Error en la Sintaxi del Fitxer .travis.yml: Assegura't que el fitxer .travis.yml estigui ben formatat.
  • Dependències No Instal·lades: Verifica que totes les dependències estiguin llistades al fitxer requirements.txt.
  • Proves Fallides: Revisa els missatges d'error de les proves per identificar i corregir els problemes.

Conclusió

La Integració Contínua és una pràctica essencial per a qualsevol projecte de programari modern. Automatitza la construcció i les proves del codi, millorant la qualitat i reduint el temps de lliurament. Amb eines com Travis CI, és fàcil configurar un entorn de CI per als teus projectes Python.

En el següent mòdul, explorarem com empaquetar i distribuir aplicacions Python, assegurant que el teu codi estigui llest per a la producció.
