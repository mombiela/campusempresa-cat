Guàrdies de ruta

Següent ►
►

Les guàrdies de ruta (route guards) són una característica poderosa d'Angular que permet controlar l'accés a les rutes de l'aplicació. Aquestes guàrdies poden ser utilitzades per protegir rutes, verificar permisos d'usuari, carregar dades abans de navegar a una ruta, entre altres funcions.

Tipus de guàrdies de ruta

Angular proporciona diversos tipus de guàrdies de ruta que es poden implementar segons les necessitats de l'aplicació:

  1. CanActivate: S'utilitza per determinar si una ruta pot ser activada.
  2. CanActivateChild: S'utilitza per determinar si les rutes filles poden ser activades.
  3. CanDeactivate: S'utilitza per determinar si es pot sortir d'una ruta.
  4. Resolve: S'utilitza per carregar dades abans d'activar una ruta.
  5. CanLoad: S'utilitza per determinar si un mòdul pot ser carregat.

Implementació de CanActivate

Pas 1: Crear un servei de guàrdia

Primer, crearem un servei que implementi la interfície CanActivate. Aquest servei contindrà la lògica per determinar si una ruta pot ser activada.

import { Injectable } from '@angular/core';
import { CanActivate, ActivatedRouteSnapshot, RouterStateSnapshot, UrlTree, Router } from '@angular/router';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class AuthGuard implements CanActivate {

  constructor(private router: Router) {}

  canActivate(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): Observable<boolean | UrlTree> | Promise<boolean | UrlTree> | boolean | UrlTree {
    const isAuthenticated = this.checkLogin();
    if (!isAuthenticated) {
      this.router.navigate(['/login']);
    }
    return isAuthenticated;
  }

  checkLogin(): boolean {
    // Aquí es pot afegir la lògica per verificar si l'usuari està autenticat
    // Per exemple, verificar un token d'autenticació
    return !!localStorage.getItem('authToken');
  }
}

Pas 2: Aplicar la guàrdia a una ruta

Després de crear el servei de guàrdia, l'aplicarem a una ruta en el fitxer de configuració de rutes.

import { NgModule } from '@angular/core';
import { RouterModule, Routes } from '@angular/router';
import { HomeComponent } from './home/home.component';
import { LoginComponent } from './login/login.component';
import { AuthGuard } from './auth.guard';

const routes: Routes = [
  { path: 'home', component: HomeComponent, canActivate: [AuthGuard] },
  { path: 'login', component: LoginComponent },
  { path: '', redirectTo: '/home', pathMatch: 'full' }
];

@NgModule({
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
})
export class AppRoutingModule { }

Pas 3: Provar la guàrdia

Ara, quan intentem accedir a la ruta /home, la guàrdia AuthGuard verificarà si l'usuari està autenticat. Si no ho està, serà redirigit a la pàgina de login.

Exercici pràctic

Exercici 1: Crear una guàrdia CanActivateChild

  1. Objectiu: Crear una guàrdia que protegeixi les rutes filles d'un mòdul.
  2. Passos:
    • Crear un servei ChildAuthGuard que implementi CanActivateChild.
    • Aplicar la guàrdia a les rutes filles en el fitxer de configuració de rutes.

Solució

Pas 1: Crear el servei ChildAuthGuard

import { Injectable } from '@angular/core';
import { CanActivateChild, ActivatedRouteSnapshot, RouterStateSnapshot, UrlTree, Router } from '@angular/router';
import { Observable } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class ChildAuthGuard implements CanActivateChild {

  constructor(private router: Router) {}

  canActivateChild(
    next: ActivatedRouteSnapshot,
    state: RouterStateSnapshot): Observable<boolean | UrlTree> | Promise<boolean | UrlTree> | boolean | UrlTree {
    const isAuthenticated = this.checkLogin();
    if (!isAuthenticated) {
      this.router.navigate(['/login']);
    }
    return isAuthenticated;
  }

  checkLogin(): boolean {
    return !!localStorage.getItem('authToken');
  }
}

Pas 2: Aplicar la guàrdia a les rutes filles

const routes: Routes = [
  {
    path: 'parent',
    component: ParentComponent,
    canActivateChild: [ChildAuthGuard],
    children: [
      { path: 'child1', component: Child1Component },
      { path: 'child2', component: Child2Component }
    ]
  },
  { path: 'login', component: LoginComponent },
  { path: '', redirectTo: '/parent', pathMatch: 'full' }
];

Resum

Les guàrdies de ruta són una eina essencial per controlar l'accés a les rutes en una aplicació Angular. Hem vist com implementar i aplicar una guàrdia CanActivate i CanActivateChild per protegir rutes i rutes filles respectivament. Aquestes guàrdies ajuden a assegurar que només els usuaris autenticats o amb els permisos necessaris puguin accedir a certes parts de l'aplicació.
