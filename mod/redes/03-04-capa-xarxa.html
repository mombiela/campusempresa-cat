Capa de Xarxa
Següent ►

Introducció

La capa de xarxa és la tercera capa del model OSI (Open Systems Interconnection) i és responsable de la determinació del camí i la commutació de paquets entre dispositius en una xarxa. Aquesta capa s'encarrega de l'encaminament dels paquets des de l'origen fins a la destinació a través de múltiples xarxes interconnectades.

Funcions Principals

Les funcions principals de la capa de xarxa inclouen:

  1. Encaminament (Routing):

    • Determina el millor camí per enviar els paquets de dades des de l'origen fins a la destinació.
    • Utilitza protocols d'encaminament com OSPF (Open Shortest Path First) i BGP (Border Gateway Protocol).

  2. Adreçament Lògic:

    • Assigna adreces IP (Internet Protocol) als dispositius de la xarxa.
    • Gestiona les adreces IP per assegurar que cada dispositiu tingui una adreça única.

  3. Fragmentació i Reassemblatge:

    • Divideix els paquets de dades en fragments més petits si són massa grans per ser transmesos.
    • Reassembla els fragments en el dispositiu de destinació.

  4. Control de Congestió:

    • Gestiona la congestió de la xarxa per evitar la saturació dels enllaços de comunicació.

Protocols de la Capa de Xarxa

Protocol IP (Internet Protocol)

El protocol IP és el protocol principal de la capa de xarxa i és responsable de l'adreçament i l'encaminament dels paquets de dades. Hi ha dues versions principals del protocol IP:

  1. IPv4 (Internet Protocol version 4):

    • Utilitza adreces de 32 bits.
    • Té un espai d'adreces limitat a 2^32 adreces (aproximadament 4.3 mil milions d'adreces).

  2. IPv6 (Internet Protocol version 6):

    • Utilitza adreces de 128 bits.
    • Té un espai d'adreces molt més gran, permetent 2^128 adreces.

Protocols d'Encaminament

Els protocols d'encaminament són utilitzats per determinar el millor camí per enviar els paquets de dades. Alguns dels protocols d'encaminament més comuns són:

  1. OSPF (Open Shortest Path First):

    • Protocol d'encaminament d'estat d'enllaç.
    • Utilitza l'algoritme de Dijkstra per trobar el camí més curt.

  2. BGP (Border Gateway Protocol):

    • Protocol d'encaminament de passarel·la exterior.
    • Utilitzat principalment per a l'encaminament entre diferents sistemes autònoms (AS).

  3. RIP (Routing Information Protocol):

    • Protocol d'encaminament de vector de distància.
    • Utilitza el nombre de salts com a mètrica per determinar el camí més curt.

Exemples Pràctics

Exemple 1: Adreçament IP

Dispositiu A: 192.168.1.1
Dispositiu B: 192.168.1.2
Màscara de Subxarxa: 255.255.255.0

En aquest exemple, els dispositius A i B estan en la mateixa subxarxa, ja que les seves adreces IP comparteixen el mateix prefix de subxarxa.

Exemple 2: Encaminament amb OSPF

Router A: 192.168.1.1
Router B: 192.168.2.1
Enllaç entre Router A i Router B: 192.168.3.0/24

En aquest exemple, els routers A i B utilitzen OSPF per determinar el camí més curt per enviar paquets entre les xarxes 192.168.1.0/24 i 192.168.2.0/24.

Exercici Pràctic

Exercici 1: Determinació del Camí

Descripció: Dibuixa una xarxa amb tres routers (A, B, C) i tres xarxes (X, Y, Z). Assigna adreces IP a cada interfície de router i determina el camí més curt des de la xarxa X fins a la xarxa Z utilitzant OSPF.

Solució:

  1. Assignació d'Adreces IP:

    • Router A: 192.168.1.1 (X), 192.168.3.1 (enllaç amb B)
    • Router B: 192.168.3.2 (enllaç amb A), 192.168.4.1 (enllaç amb C)
    • Router C: 192.168.4.2 (enllaç amb B), 192.168.2.1 (Z)

  2. Determinació del Camí:

    • Utilitzant OSPF, el camí més curt des de X (192.168.1.0/24) fins a Z (192.168.2.0/24) és:
      • X -> Router A -> Router B -> Router C -> Z

Resum

La capa de xarxa és essencial per a l'encaminament i l'adreçament de paquets en una xarxa. Els protocols com IP, OSPF i BGP juguen un paper crucial en la determinació del camí i la gestió de la comunicació entre dispositius. Comprendre aquestes funcions i protocols és fonamental per a qualsevol professional de xarxes.
