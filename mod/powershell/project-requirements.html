Requisits del Projecte
Construir el Projecte ►

En aquesta secció, cobrirem els requisits essencials necessaris per començar un projecte de PowerShell. Entendre aquests requisits és crucial tant per a principiants com per a usuaris avançats per assegurar un flux de treball suau i eficient.

Conceptes Clau

  • Entorn PowerShell: Configuració i configuració de l'entorn PowerShell.
  • Programari i Eines: Programari i eines necessàries per a l'scripting de PowerShell.
  • Permisos i Seguretat: Entendre els permisos i les configuracions de seguretat.
  • Planificació del Projecte: Planificació i definició de l'abast del teu projecte PowerShell.

Entorn PowerShell

Abans de començar a escriure scripts en PowerShell, has d'assegurar-te que el teu entorn està correctament configurat.

Windows PowerShell vs. PowerShell Core

PowerShell ve en dues versions principals: Windows PowerShell i PowerShell Core. Entendre les diferències entre elles és crucial per seleccionar la correcta per al teu projecte.

| Característica | Windows PowerShell | PowerShell Core | |------------------------|--------------------|-----------------| | Plataforma | Només Windows | Multiplataforma (Windows, macOS, Linux) | | Versió més recent | 5.1 | 7.x | | .NET Framework | .NET Framework | .NET Core/.NET 5+ | | Compatibilitat de Mòduls | Alta | Variable |

Configuració de PowerShell

  1. Instal·lar PowerShell:

    • Per a Windows PowerShell, està preinstal·lat en sistemes Windows.
    • Per a PowerShell Core, descarrega i instal·la des del repositori oficial de GitHub.

  2. Verificar la Instal·lació:

    $PSVersionTable.PSVersion

  3. Actualitzar PowerShell:

    • Utilitza la següent comanda per actualitzar PowerShell Core: 
      Install-Module -Name PowerShellGet -Force -AllowClobber
Update-Module -Name PowerShellGet

Programari i Eines

Per millorar la teva experiència d'scripting en PowerShell, podries necessitar programari i eines addicionals.

  • Entorn de Desenvolupament Integrat (IDE):

    • Visual Studio Code: Un editor de codi popular i gratuït amb suport per a extensions de PowerShell.
    • PowerShell Integrated Scripting Environment (ISE): Editor integrat per a Windows PowerShell.

  • Control de Versions:

    • Git: Per gestionar els teus scripts de PowerShell i col·laborar amb altres.

  • Mòduls i Paquets:

    • PowerShell Gallery: Un repositori de mòduls i scripts de PowerShell.
    • Exemple per instal·lar un mòdul: 
      Install-Module -Name Az -AllowClobber

Permisos i Seguretat

Els scripts de PowerShell poden realitzar operacions potents, així que entendre els permisos i la seguretat és essencial.

  • Política d'Execució:

    • Determina quins scripts es poden executar al teu sistema.
    • Polítiques comunes:
      • Restricted: No es permeten scripts.
      • RemoteSigned: Els scripts descarregats d'internet han d'estar signats.
      • Unrestricted: Es poden executar tots els scripts.
    • Exemple per establir la política d'execució: 
      Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

  • Executar com a Administrador:

    • Alguns scripts requereixen permisos elevats.
    • Per executar PowerShell com a administrador, fes clic dret a la icona de PowerShell i selecciona "Executar com a administrador".

Planificació del Projecte

Una bona planificació és clau per a un projecte de PowerShell exitós.

  • Definir Objectius: Especifica clarament què vols aconseguir amb el teu script.
  • Identificar Requisits: Llista tots els inputs, outputs i dependències necessàries.
  • Dissenyar l'Script: Planifica l'estructura i el flux del teu script.
  • Provar i Depurar: Destina temps per provar i depurar el teu script.

Exemple de Pla de Projecte

  1. Objectiu: Automatitzar la creació de comptes d'usuari a Active Directory.
  2. Requisits:
    • Input: Fitxer CSV amb detalls dels usuaris.
    • Output: Nous comptes d'usuari a Active Directory.
    • Dependències: Mòdul d'Active Directory.
  3. Disseny de l'Script:
    • Importar detalls dels usuaris des del CSV.
    • Recórrer cada usuari i crear un compte.
    • Gestionar errors i registrar resultats.
  4. Proves:
    • Provar amb un petit subconjunt d'usuaris.
    • Validar la creació dels comptes.

Conclusió

Entendre els requisits del projecte és el primer pas cap a un scripting de PowerShell exitós. Configurant l'entorn adequat, utilitzant les eines apropiades, assegurant la seguretat i planificant el teu projecte meticulosament, pots optimitzar el teu flux de treball i assolir els teus objectius d'scripting de manera eficient.
