PowerShell i la Integració/Desplegament Continu (CI/CD)

Introducció a CI/CD

  • Integració Contínua (CI): La pràctica de fusionar totes les còpies de treball dels desenvolupadors a una línia principal compartida diverses vegades al dia.
  • Desplegament Continu (CD): La pràctica de desplegar automàticament cada canvi que passa totes les etapes de la teva pipeline de producció a producció.

Per què utilitzar PowerShell per a CI/CD?

  • Automatització: Els scripts de PowerShell poden automatitzar tasques repetitives.
  • Integració: PowerShell pot interactuar amb diverses eines de CI/CD com Jenkins, Azure DevOps i GitHub Actions.
  • Flexibilitat: PowerShell es pot utilitzar en diferents entorns (Windows, Linux, macOS).

Configuració del teu entorn

Prerequisits

  • Instal·lar PowerShell Core (si no està ja instal·lat).
  • Familiaritat amb comandaments bàsics de PowerShell.
  • Accés a una eina de CI/CD (per exemple, Jenkins, Azure DevOps, GitHub Actions).

Instal·lació de mòduls de PowerShell

# Instal·lar el mòdul d'Azure DevOps
Install-Module -Name AzureDevOps -Scope CurrentUser

# Instal·lar el mòdul de Jenkins
Install-Module -Name Jenkins -Scope CurrentUser

Tasques bàsiques de CI/CD amb PowerShell

Executar proves unitàries

  • Utilitza PowerShell per executar proves unitàries com a part de la pipeline de CI.
# Exemple: Executar proves de Pester
Invoke-Pester -Path .\Tests

Construir projectes

  • Automatitza el procés de construcció utilitzant scripts de PowerShell.
# Exemple: Construir un projecte .NET
dotnet build .\MyProject.csproj

Desplegar aplicacions

  • Utilitza PowerShell per desplegar aplicacions a diferents entorns.
# Exemple: Desplegar una aplicació web a Azure
az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\MyApp.zip

Integració de PowerShell amb eines de CI/CD

Jenkins

  • Utilitza scripts de PowerShell en pipelines de Jenkins.
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                powershell 'dotnet build .\\MyProject.csproj'
            }
        }
        stage('Test') {
            steps {
                powershell 'Invoke-Pester -Path .\\Tests'
            }
        }
        stage('Deploy') {
            steps {
                powershell 'az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\\MyApp.zip'
            }
        }
    }
}

Azure DevOps

  • Utilitza tasques de PowerShell en Azure Pipelines.
trigger:
- main

pool:
  vmImage: 'windows-latest'

steps:
- task: PowerShell@2
  inputs:
    targetType: 'inline'
    script: 'dotnet build .\MyProject.csproj'

- task: PowerShell@2
  inputs:
    targetType: 'inline'
    script: 'Invoke-Pester -Path .\Tests'

- task: AzureCLI@2
  inputs:
    azureSubscription: 'MyAzureSubscription'
    scriptType: 'ps'
    scriptLocation: 'inlineScript'
    inlineScript: 'az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\MyApp.zip'

GitHub Actions

  • Utilitza PowerShell en workflows de GitHub Actions.
name: CI/CD Pipeline

on: [push]

jobs:
  build:
    runs-on: windows-latest

    steps:
    - uses: actions/checkout@v2

    - name: Build
      run: dotnet build .\MyProject.csproj
      shell: pwsh

    - name: Test
      run: Invoke-Pester -Path .\Tests
      shell: pwsh

    - name: Deploy
      run: az webapp deploy --resource-group MyResourceGroup --name MyWebApp --src-path .\MyApp.zip
      shell: pwsh

CI/CD avançat amb PowerShell

Mòduls personalitzats de PowerShell per a CI/CD

  • Crea mòduls personalitzats de PowerShell per encapsular la lògica de CI/CD.
# Exemple: Mòdul personalitzat per al desplegament
function Deploy-WebApp {
    param (
        [string]$ResourceGroup,
        [string]$WebAppName,
        [string]$SourcePath
    )
    az webapp deploy --resource-group $ResourceGroup --name $WebAppName --src-path $SourcePath
}

Export-ModuleMember -Function Deploy-WebApp

Gestió d'errors i registre

  • Implementa una gestió robusta d'errors i registre en scripts de PowerShell.
try {
    # Exemple: Procés de construcció
    dotnet build .\MyProject.csproj
} catch {
    Write-Error "La construcció ha fallat: $_"
    exit 1
}

try {
    # Exemple: Procés de desplegament
    Deploy-WebApp -ResourceGroup "MyResourceGroup" -WebAppName "MyWebApp" -SourcePath ".\MyApp.zip"
} catch {
    Write-Error "El desplegament ha fallat: $_"
    exit 1
}

Conclusió

PowerShell és una eina poderosa per automatitzar pipelines de CI/CD. Integrant scripts de PowerShell amb eines de CI/CD com Jenkins, Azure DevOps i GitHub Actions, pots optimitzar els teus processos de desenvolupament i desplegament. Tant si estàs executant proves, construint projectes o desplegant aplicacions, PowerShell proporciona la flexibilitat i el control necessaris per gestionar els teus workflows de CI/CD de manera efectiva.
