Monitorització i auditoria
Següent ►

Introducció

En aquest tema, aprendrem com monitoritzar i auditar l'ús de Power BI per assegurar-nos que les dades es gestionen de manera segura i eficient. La monitorització i l'auditoria són components essencials per mantenir la integritat i la seguretat de les dades, així com per garantir el compliment de les polítiques internes i les regulacions externes.

Conceptes clau

  1. Monitorització: Seguiment continu de l'activitat i el rendiment dels recursos de Power BI.
  2. Auditoria: Registre detallat de les accions i esdeveniments que ocorren dins de Power BI per a la seva revisió i anàlisi posterior.

Eines de monitorització i auditoria en Power BI

1. Power BI Activity Log

El registre d'activitat de Power BI proporciona informació detallada sobre les accions realitzades pels usuaris dins del servei de Power BI. Aquest registre és essencial per a la monitorització i l'auditoria.

Accions comunes registrades:

  • Visualització de informes
  • Creació i modificació de contingut
  • Compartició de contingut
  • Exportació de dades

Exemple de registre d'activitat:

{
  "RecordType": "PowerBI",
  "CreationTime": "2023-10-01T12:34:56Z",
  "Operation": "ViewReport",
  "UserId": "user@example.com",
  "ReportName": "Informe de Vendes",
  "WorkspaceName": "Vendes"
}

2. Power BI Admin Portal

El portal d'administració de Power BI proporciona eines per monitoritzar l'ús i el rendiment dels recursos de Power BI.

Funcionalitats clau:

  • Mètriques d'ús: Informació sobre l'ús dels informes, quadres de comandament i altres recursos.
  • Rendiment: Estadístiques sobre el rendiment de les consultes i els temps de càrrega.
  • Alertes: Configuració d'alertes per a esdeveniments específics.

3. Azure Monitor

Azure Monitor pot ser utilitzat per recollir i analitzar dades de telemetria de Power BI, proporcionant una visió més completa del rendiment i l'ús.

Funcionalitats clau:

  • Logs: Recollida de logs detallats d'activitat.
  • Mètriques: Monitorització de mètriques de rendiment en temps real.
  • Alertes: Configuració d'alertes basades en condicions específiques.

Exercici pràctic

Objectiu:

Configurar una alerta en el portal d'administració de Power BI per notificar quan un informe específic és visualitzat més de 100 vegades en un dia.

Passos:

  1. Accedir al portal d'administració de Power BI:

    • Navega a Power BI Admin Portal.

  2. Navegar a la secció de mètriques d'ús:

    • Selecciona "Usage metrics" al menú lateral.

  3. Configurar una alerta:

    • Selecciona l'informe desitjat.
    • Configura una alerta per a "Views per day" amb una condició de "greater than 100".

Solució:

1. Accedeix al portal d'administració de Power BI.
2. Navega a "Usage metrics".
3. Selecciona l'informe "Informe de Vendes".
4. Configura una alerta:
   - Mètrica: Views per day
   - Condició: greater than 100
   - Notificació: Correu electrònic a admin@example.com

Errors comuns i consells

Errors comuns:

  • No configurar alertes adequades: Assegura't que les alertes estan configurades per a les mètriques més rellevants.
  • Ignorar els logs d'activitat: Revisa regularment els logs d'activitat per detectar comportaments anòmals.

Consells:

  • Automatitza les revisions: Utilitza eines com Azure Monitor per automatitzar la recollida i l'anàlisi de dades.
  • Forma als usuaris: Educa els usuaris sobre les millors pràctiques de seguretat i compliment.

Conclusió

La monitorització i l'auditoria són components essencials per mantenir la seguretat i l'eficiència en l'ús de Power BI. Utilitzant les eines disponibles, com el registre d'activitat de Power BI, el portal d'administració i Azure Monitor, pots assegurar-te que les teves dades estan protegides i que el rendiment dels teus recursos és òptim.

En el següent tema, explorarem les millors pràctiques de governança de Power BI per assegurar-nos que les dades es gestionen de manera coherent i segura a tota l'organització.
