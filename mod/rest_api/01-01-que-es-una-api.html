Què és una API?
Següent ►

Introducció

Una API (Application Programming Interface) és un conjunt de regles i protocols que permeten a diferents aplicacions comunicar-se entre elles. Les APIs defineixen la manera en què les peticions i respostes han de ser estructurades, facilitant la integració i la interoperabilitat entre sistemes diversos.

Conceptes Clau

  1. Interfície: És el punt de contacte entre dues entitats, en aquest cas, entre dues aplicacions. Les APIs actuen com a interfícies que permeten aquesta interacció.
  2. Protocol: És un conjunt de regles que defineixen com es realitza la comunicació. En el context de les APIs, els protocols més comuns són HTTP/HTTPS.
  3. Petitions i Respostes: Les APIs funcionen mitjançant peticions (requests) que una aplicació fa a una altra, i respostes (responses) que l'aplicació receptora retorna.

Tipus d'APIs

  1. APIs de Sistema: Permeten la comunicació entre el sistema operatiu i les aplicacions.
  2. APIs de Biblioteca: Proporcionen funcionalitats específiques a través de biblioteques de programari.
  3. APIs de Serveis Web: Permeten la comunicació entre aplicacions a través d'Internet. Les APIs RESTful són un tipus d'API de serveis web.

Exemples d'Ús d'APIs

  1. Integració de Serveis: Les APIs permeten integrar serveis de tercers, com ara sistemes de pagament, serveis de mapes, etc.
  2. Automatització de Tasques: Les APIs poden ser utilitzades per automatitzar processos repetitius, com ara l'extracció de dades o l'enviament de correus electrònics.
  3. Aplicacions Mòbils: Les aplicacions mòbils sovint utilitzen APIs per comunicar-se amb servidors i obtenir dades en temps real.

Components d'una API

  1. Endpoint: És l'URL a la qual es fa la petició. Cada endpoint representa una funcionalitat específica de l'API.
  2. Mètodes HTTP: Defineixen l'acció que es vol realitzar. Els mètodes més comuns són GET, POST, PUT, DELETE.
  3. Paràmetres: Són dades addicionals que es poden enviar amb la petició per especificar detalls addicionals.
  4. Codis d'Estat: Indiquen el resultat de la petició. Els codis més comuns són 200 (èxit), 404 (no trobat), 500 (error del servidor).

Exemple Pràctic

A continuació, es mostra un exemple d'una petició HTTP a una API RESTful per obtenir informació d'un usuari:

Petició

GET /users/123 HTTP/1.1
Host: api.example.com
Authorization: Bearer token123

Resposta

{
  "id": 123,
  "name": "Joan",
  "email": "joan@example.com"
}

Exercici Pràctic

Exercici 1

Objectiu: Realitzar una petició GET a una API pública per obtenir informació sobre un usuari.

  1. Tria una API pública, com ara JSONPlaceholder.
  2. Realitza una petició GET a l'endpoint /users/1.
  3. Analitza la resposta obtinguda.

Solució

GET /users/1 HTTP/1.1
Host: jsonplaceholder.typicode.com

Resposta esperada:

{
  "id": 1,
  "name": "Leanne Graham",
  "username": "Bret",
  "email": "Sincere@april.biz",
  ...
}

Conclusió

En aquesta secció hem après què és una API, els seus components principals i com s'utilitzen en diferents contextos. Les APIs són fonamentals per a la integració i la comunicació entre aplicacions, facilitant la interoperabilitat i l'automatització de processos. En el següent tema, explorarem la història i l'evolució de les APIs per entendre com han arribat a ser tan importants en el desenvolupament de programari modern.
