Exercicis del Model TCP/IP

Següent ►
►

En aquesta secció, treballarem amb exercicis pràctics per reforçar els conceptes apresos sobre el model TCP/IP. Cada exercici inclou una explicació detallada i la seva solució per assegurar que comprenguis completament els conceptes.

Exercici 1: Identificació de Capes TCP/IP

Enunciat

Llegeix la següent descripció d'un procés de comunicació i identifica a quina capa del model TCP/IP pertany cada acció descrita:

  1. Un navegador web envia una sol·licitud HTTP per obtenir una pàgina web.
  2. La sol·licitud HTTP es divideix en segments i s'assigna un número de port.
  3. Els segments es converteixen en paquets IP amb adreces IP d'origen i destinació.
  4. Els paquets IP es converteixen en trames i s'envien a través de la xarxa física.

Solució

  1. Capa d'Aplicació: El navegador web envia una sol·licitud HTTP.
  2. Capa de Transport: La sol·licitud HTTP es divideix en segments i s'assigna un número de port.
  3. Capa d'Internet: Els segments es converteixen en paquets IP amb adreces IP d'origen i destinació.
  4. Capa d'Accés a la Xarxa: Els paquets IP es converteixen en trames i s'envien a través de la xarxa física.

Exercici 2: Adreçament IP

Enunciat

Suposa que tens la següent adreça IP: 192.168.1.10/24. Respon les següents preguntes:

  1. Quina és la màscara de subxarxa?
  2. Quina és l'adreça de xarxa?
  3. Quina és l'adreça de broadcast?
  4. Quin és el rang d'adreces IP disponibles per als hosts?

Solució

  1. Màscara de subxarxa: 255.255.255.0
  2. Adreça de xarxa: 192.168.1.0
  3. Adreça de broadcast: 192.168.1.255
  4. Rang d'adreces IP disponibles per als hosts: 192.168.1.1 - 192.168.1.254

Exercici 3: Protocols de Transport

Enunciat

Classifica els següents protocols com a TCP o UDP:

  1. HTTP
  2. DNS
  3. FTP
  4. DHCP
  5. SMTP

Solució

  1. HTTP: TCP
  2. DNS: UDP (tot i que també pot utilitzar TCP en alguns casos)
  3. FTP: TCP
  4. DHCP: UDP
  5. SMTP: TCP

Exercici 4: Anàlisi de Tràfic de Xarxa

Enunciat

Utilitza una eina d'anàlisi de tràfic de xarxa com Wireshark per capturar i analitzar el tràfic de xarxa. Completa les següents tasques:

  1. Captura el tràfic mentre accedeixes a una pàgina web.
  2. Identifica un paquet HTTP en la captura.
  3. Determina l'adreça IP d'origen i destinació del paquet HTTP.
  4. Identifica el número de port d'origen i destinació del paquet HTTP.

Solució

  1. Captura de tràfic: Obre Wireshark i comença una nova captura mentre accedeixes a una pàgina web.
  2. Identificació del paquet HTTP: Filtra els paquets HTTP utilitzant el filtre http.
  3. Adreça IP d'origen i destinació: Selecciona un paquet HTTP i revisa les adreces IP en la secció d'IP del paquet.
  4. Número de port d'origen i destinació: Revisa la secció de TCP del paquet per trobar els números de port.

Exercici 5: Configuració de Xarxa

Enunciat

Configura manualment una adreça IP estàtica en un sistema operatiu de la teva elecció. Completa les següents tasques:

  1. Assigna l'adreça IP 192.168.1.100 amb una màscara de subxarxa 255.255.255.0.
  2. Configura la porta d'enllaç per defecte com 192.168.1.1.
  3. Configura el servidor DNS com 8.8.8.8.

Solució

Windows:

  1. Obre el "Centre de xarxes i recursos compartits".
  2. Fes clic a "Canviar la configuració de l'adaptador".
  3. Fes clic dret a la connexió de xarxa i selecciona "Propietats".
  4. Selecciona "Protocol d'Internet versió 4 (TCP/IPv4)" i fes clic a "Propietats".
  5. Selecciona "Utilitza la següent adreça IP" i introdueix 192.168.1.100.
  6. Introdueix la màscara de subxarxa 255.255.255.0.
  7. Introdueix la porta d'enllaç per defecte 192.168.1.1.
  8. Selecciona "Utilitza les següents adreces de servidor DNS" i introdueix 8.8.8.8.

Linux:

  1. Edita el fitxer /etc/network/interfaces (pot variar segons la distribució).
  2. Afegeix les següents línies: 
    auto eth0
iface eth0 inet static
    address 192.168.1.100
    netmask 255.255.255.0
    gateway 192.168.1.1
    dns-nameservers 8.8.8.8
  3. Reinicia la interfície de xarxa amb sudo ifdown eth0 && sudo ifup eth0.

Conclusió

En aquesta secció, hem treballat amb diversos exercicis pràctics per reforçar els conceptes del model TCP/IP. Hem identificat capes, treballat amb adreces IP, classificat protocols de transport, analitzat tràfic de xarxa i configurat adreces IP estàtiques. Aquests exercicis t'ajudaran a comprendre millor com funciona el model TCP/IP en la pràctica.
