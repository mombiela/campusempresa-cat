Introducció a WCAG

Següent ►
►

En aquesta secció, explorarem les Directrius d'Accessibilitat per al Contingut Web (WCAG), que són un conjunt de recomanacions per fer que el contingut web sigui més accessible per a tothom, incloses les persones amb discapacitats. Les WCAG són essencials per garantir que els llocs web siguin utilitzables per una audiència més àmplia.

Què són les WCAG?

Les WCAG són un conjunt de directrius desenvolupades pel World Wide Web Consortium (W3C) per ajudar els dissenyadors i desenvolupadors web a crear contingut accessible. Aquestes directrius es basen en quatre principis fonamentals:

  1. Perceptible: La informació i els components de la interfície d'usuari han de ser presentats als usuaris de manera que puguin percebre'ls.
  2. Operable: Els components de la interfície d'usuari i la navegació han de ser operables.
  3. Comprensible: La informació i el funcionament de la interfície d'usuari han de ser comprensibles.
  4. Robust: El contingut ha de ser prou robust per ser interpretat de manera fiable per una àmplia varietat d'agents d'usuari, incloses les tecnologies assistencials.

Nivells de Conformitat

Les WCAG defineixen tres nivells de conformitat per mesurar l'accessibilitat d'un lloc web:

  • Nivell A: El nivell mínim d'accessibilitat. Compleix amb els criteris essencials per a l'accessibilitat.
  • Nivell AA: Un nivell intermedi que aborda els obstacles més comuns per a l'accessibilitat.
  • Nivell AAA: El nivell més alt d'accessibilitat, que proporciona la màxima accessibilitat possible.

Estructura de les WCAG

Les WCAG estan organitzades en principis, pautes i criteris de conformitat:

  • Principis: Els quatre principis fonamentals (Perceptible, Operable, Comprensible, Robust).
  • Pautes: Cada principi conté diverses pautes que proporcionen els objectius generals per a l'accessibilitat.
  • Criteris de Conformitat: Cada pauta inclou criteris de conformitat que són mesurables i que determinen si s'ha assolit un nivell específic d'accessibilitat.

Exemples Pràctics

Exemple de Perceptible

<img src="imatge.jpg" alt="Descripció de la imatge">
  • Explicació: L'atribut alt proporciona una descripció textual de la imatge, fent-la accessible per a usuaris amb discapacitats visuals que utilitzen lectors de pantalla.

Exemple d'Operable

<button onclick="submitForm()">Enviar</button>
  • Explicació: Assegura't que els botons i altres elements interactius siguin accessibles mitjançant el teclat, no només amb el ratolí.

Exercici Pràctic

Exercici: Revisa una pàgina web i identifica almenys tres elements que podrien millorar-se per complir amb les WCAG. Proposa solucions per a cada element.

Solució:

  1. Imatge sense text alternatiu: Afegeix un atribut alt amb una descripció adequada.
  2. Botó que només es pot activar amb el ratolí: Assegura't que el botó es pugui activar amb el teclat afegint l'atribut tabindex.
  3. Text amb contrast insuficient: Augmenta el contrast entre el text i el fons per millorar la llegibilitat.

Conclusió

Les WCAG són una eina essencial per crear llocs web accessibles. Comprendre i aplicar aquestes directrius ajuda a garantir que el contingut web sigui accessible per a tothom, independentment de les seves capacitats. En el proper mòdul, explorarem com les persones amb discapacitats utilitzen la web i les tecnologies assistencials que els ajuden.
