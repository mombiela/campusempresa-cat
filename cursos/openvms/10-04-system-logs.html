Logs del Sistema i Diagnòstics

Següent ►
►

En aquest tema, explorarem com utilitzar els logs del sistema a OpenVMS per diagnosticar problemes i monitoritzar el rendiment del sistema. Els logs són una eina essencial per a qualsevol administrador de sistemes, ja que proporcionen informació detallada sobre el funcionament intern del sistema operatiu i les aplicacions que s'hi executen.

Objectius del Tema

  • Entendre la importància dels logs del sistema.
  • Aprendre a accedir i interpretar els logs del sistema a OpenVMS.
  • Utilitzar eines i comandes per gestionar i analitzar els logs.
  • Diagnosticar problemes comuns utilitzant els logs.

  1. Importància dels Logs del Sistema

Els logs del sistema són registres detallats d'esdeveniments que ocorren dins del sistema operatiu i les aplicacions. Aquests registres poden incloure informació sobre errors, advertències, esdeveniments informatius i altres activitats del sistema. Els logs són crucials per a:

  • Diagnòstic de Problemes: Identificar i resoldre errors i problemes de rendiment.
  • Seguretat: Monitoritzar activitats sospitoses i detectar possibles amenaces.
  • Auditoria: Mantenir un registre d'activitats per a complir amb regulacions i polítiques internes.
  • Optimització: Analitzar el rendiment del sistema i identificar àrees de millora.

  1. Accés als Logs del Sistema a OpenVMS

A OpenVMS, els logs del sistema es poden trobar en diversos fitxers i ubicacions. Alguns dels logs més importants inclouen:

  • Operator Log (OPERATOR.LOG): Registra esdeveniments del sistema i missatges d'operador.
  • Accounting Log (ACCOUNTING.LOG): Registra informació sobre l'ús de recursos per part dels usuaris.
  • Error Log (ERRLOG.SYS): Registra errors de maquinari i altres esdeveniments crítics.

Comandes per Accedir als Logs

  • OPCOM: Utilitzada per veure missatges d'operador en temps real.

    $ REPLY/ENABLE
$ REPLY/LOG

  • ANALYZE/ERROR_LOG: Utilitzada per analitzar el log d'errors.

    $ ANALYZE/ERROR_LOG

  • SHOW ACCOUNTING: Utilitzada per veure registres d'ús de recursos.

    $ SHOW ACCOUNTING

  1. Interpretació dels Logs

Operator Log (OPERATOR.LOG)

Aquest log conté missatges d'operador que poden incloure informació sobre esdeveniments del sistema, com ara arrencades, parades, errors de dispositius, etc. Exemple d'una entrada típica:

%%%%%%%%%%%  OPCOM  12-DEC-2023 12:34:56.78  %%%%%%%%%%%
Message from user SYSTEM on NODE
%SHUTDOWN-I-SHUTDOWN, system shutdown has been initiated

Error Log (ERRLOG.SYS)

Aquest log conté informació detallada sobre errors de maquinari i altres esdeveniments crítics. Exemple d'una entrada típica:

%%%%%%%%%%%  OPCOM  12-DEC-2023 12:34:56.78  %%%%%%%%%%%
Message from user SYSTEM on NODE
%ERRLOG-F-HARDWARE, hardware error detected on device DKA0:

  1. Eines i Comandes per Gestionar i Analitzar els Logs

OPCOM

OPCOM és una eina que permet veure missatges d'operador en temps real. Per habilitar OPCOM:

$ REPLY/ENABLE
$ REPLY/LOG

ANALYZE/ERROR_LOG

Aquesta comanda s'utilitza per analitzar el log d'errors. Exemple d'ús:

$ ANALYZE/ERROR_LOG

SHOW ACCOUNTING

Aquesta comanda mostra registres d'ús de recursos:

$ SHOW ACCOUNTING

  1. Diagnòstic de Problemes Comuns

Problema: Error de Maquinari

Símptoma: El sistema reporta errors de dispositiu.

Solució:

  1. Utilitza ANALYZE/ERROR_LOG per revisar el log d'errors.
  2. Identifica el dispositiu afectat.
  3. Consulta la documentació del maquinari per a possibles solucions.

Problema: Rendiment Lent

Símptoma: El sistema funciona més lent del normal.

Solució:

  1. Revisa l'ACCOUNTING.LOG per identificar processos que consumeixen molts recursos.
  2. Utilitza SHOW SYSTEM per veure l'ús actual de recursos.
  3. Optimitza o finalitza processos que consumeixen molts recursos.

Exercicis Pràctics

Exercici 1: Accedir i Visualitzar el Operator Log

  1. Habilita OPCOM per veure missatges d'operador en temps real.
  2. Genera un esdeveniment (per exemple, inicia una parada del sistema) i observa el missatge al log.

Solució:

$ REPLY/ENABLE
$ REPLY/LOG
$ @SYS$SYSTEM:SHUTDOWN

Exercici 2: Analitzar el Error Log

  1. Utilitza ANALYZE/ERROR_LOG per revisar el log d'errors.
  2. Identifica un error de maquinari i descriu els passos per solucionar-lo.

Solució:

$ ANALYZE/ERROR_LOG

Exercici 3: Revisar el Accounting Log

  1. Utilitza SHOW ACCOUNTING per veure registres d'ús de recursos.
  2. Identifica un procés que consumeix molts recursos i proposa una solució per optimitzar-lo.

Solució:

$ SHOW ACCOUNTING

Resum

En aquest tema, hem après la importància dels logs del sistema a OpenVMS i com accedir-hi i interpretar-los. Hem explorat diverses eines i comandes per gestionar i analitzar els logs, i hem vist com utilitzar aquesta informació per diagnosticar problemes comuns. Els logs del sistema són una eina poderosa per a qualsevol administrador de sistemes, i la seva correcta utilització pot millorar significativament la gestió i el rendiment del sistema.
