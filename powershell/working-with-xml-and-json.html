Introducció

En aquesta secció, explorarem com treballar amb els formats de dades XML i JSON a PowerShell. Aquests formats són àmpliament utilitzats per a l'intercanvi de dades i fitxers de configuració. Entendre com manipular-los és crucial per automatitzar tasques i integrar sistemes.

XML a PowerShell

Conceptes Clau

  • XML (eXtensible Markup Language): Un llenguatge de marques que defineix un conjunt de regles per codificar documents en un format que és llegible tant per humans com per màquines.
  • XML Document Object Model (DOM): Una interfície de programació per a documents XML que permet als programes accedir i actualitzar dinàmicament el contingut, l'estructura i l'estil dels documents XML.

Llegir XML

Per llegir dades XML a PowerShell, pots utilitzar el cmdlet Get-Content per carregar el fitxer XML i després convertir-lo a un objecte XML.

# Carregar fitxer XML
[xml]$xmlData = Get-Content -Path "C:\path\to\your\file.xml"

# Accedir a elements XML
$xmlData.ElementName

Modificar XML

Pots modificar dades XML navegant pels seus nodes i actualitzant els valors.

# Modificar un element XML
$xmlData.ElementName.SubElementName = "New Value"

# Desar els canvis al fitxer
$xmlData.Save("C:\path\to\your\file.xml")

Crear XML

Crear un document XML des de zero implica crear una declaració XML i afegir elements.

# Crear un nou document XML
$xmlDoc = New-Object System.Xml.XmlDocument
$root = $xmlDoc.CreateElement("RootElement")
$xmlDoc.AppendChild($root)

# Afegir elements
$element = $xmlDoc.CreateElement("ChildElement")
$element.InnerText = "Element Value"
$root.AppendChild($element)

# Desar el document XML
$xmlDoc.Save("C:\path\to\your\newfile.xml")

JSON a PowerShell

Conceptes Clau

  • JSON (JavaScript Object Notation): Un format d'intercanvi de dades lleuger que és fàcil de llegir i escriure per a humans i fàcil de parsejar i generar per a màquines.
  • PSCustomObject: Un tipus d'objecte de PowerShell que sovint s'utilitza per crear dades estructurades que es poden convertir fàcilment a JSON.

Llegir JSON

Per llegir dades JSON, utilitza el cmdlet Get-Content per carregar el fitxer JSON i després converteix-lo a un objecte de PowerShell utilitzant ConvertFrom-Json.

# Carregar fitxer JSON
$jsonData = Get-Content -Path "C:\path\to\your\file.json" | ConvertFrom-Json

# Accedir a propietats JSON
$jsonData.PropertyName

Modificar JSON

Pots modificar dades JSON actualitzant les propietats de l'objecte de PowerShell i després convertint-lo de nou a JSON.

# Modificar una propietat JSON
$jsonData.PropertyName = "New Value"

# Convertir de nou a JSON i desar
$jsonData | ConvertTo-Json | Set-Content -Path "C:\path\to\your\file.json"

Crear JSON

Crear dades JSON implica crear un PSCustomObject i després convertir-lo a JSON.

# Crear un nou objecte JSON
$jsonObject = [PSCustomObject]@{
    Property1 = "Value1"
    Property2 = "Value2"
}

# Convertir a JSON i desar
$jsonObject | ConvertTo-Json | Set-Content -Path "C:\path\to\your\newfile.json"

Comparant XML i JSON

| Característica | XML | JSON | |----------------|-------------------------------------|-----------------------------------| | Sintaxi | Llenguatge de marques amb etiquetes | Parells clau-valor | | Llegibilitat | Verbós, llegible per a humans | Concís, llegible per a humans | | Parseig | Requereix un parser XML | Fàcilment parsejat amb funcions integrades | | Usos | Fitxers de configuració, emmagatzematge de documents | Intercanvi de dades, APIs |

Conclusió

Treballar amb XML i JSON a PowerShell és essencial per automatitzar l'intercanvi de dades i la gestió de configuracions. Entenent com llegir, modificar i crear aquests formats de dades, pots millorar la funcionalitat i la interoperabilitat dels teus scripts. Practica amb els exemples proporcionats i explora més per dominar aquestes habilitats.
