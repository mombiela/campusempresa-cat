Què és DevOps?

DevOps és un conjunt de pràctiques que combina el desenvolupament de programari (Dev) i les operacions de TI (Ops). Té com a objectiu escurçar el cicle de vida del desenvolupament de sistemes i proporcionar lliurament continu amb alta qualitat de programari.

Conceptes Clau de DevOps

  • Col·laboració: Trencar les barreres entre els equips de desenvolupament i operacions.
  • Automatització: Automatitzar tasques repetitives per augmentar l'eficiència i reduir errors.
  • Integració Contínua (CI): Integrar freqüentment els canvis de codi en un repositori compartit.
  • Lliurament Continu (CD): Assegurar que els canvis de codi es proven automàticament i es preparen per al seu llançament a producció.
  • Monitorització i Registre: Monitoritzar i registrar contínuament per assegurar la salut i el rendiment del sistema.

Per què Utilitzar PowerShell en DevOps?

PowerShell és un llenguatge de script i un marc d'automatització potent de Microsoft. S'utilitza àmpliament en DevOps per automatitzar tasques, gestionar configuracions i orquestrar fluxos de treball.

Beneficis d'Utilitzar PowerShell en DevOps

  • Multiplataforma: PowerShell Core està disponible en Windows, macOS i Linux.
  • Integració: S'integra perfectament amb diverses eines i serveis com Azure, AWS, Docker i més.
  • Automatització: Simplifica l'automatització de tasques complexes.
  • Extensibilitat: Suporta mòduls i scripts personalitzats.

Configuració de PowerShell per a DevOps

Abans de submergir-te en les pràctiques de DevOps amb PowerShell, necessites configurar el teu entorn.

Instal·lació de PowerShell

  • Windows: PowerShell està preinstal·lat a Windows. Per a la versió més recent, instal·la PowerShell Core des del lloc oficial.
  • macOS/Linux: Instal·la PowerShell Core utilitzant el gestor de paquets del teu sistema operatiu.

Exemple: Instal·lació de PowerShell Core a Ubuntu

# Actualitza la llista de paquets
sudo apt-get update

# Instal·la els paquets necessaris
sudo apt-get install -y wget apt-transport-https software-properties-common

# Descarrega i instal·la les claus GPG del repositori de Microsoft
wget -q https://packages.microsoft.com/config/ubuntu/20.04/packages-microsoft-prod.deb

# Registra les claus GPG del repositori de Microsoft
sudo dpkg -i packages-microsoft-prod.deb

# Actualitza la llista de paquets després d'afegir el repositori de Microsoft
sudo apt-get update

# Instal·la PowerShell
sudo apt-get install -y powershell

# Inicia PowerShell
pwsh

Comandes Bàsiques de PowerShell per a DevOps

Navegació pel Sistema de Fitxers

# Llistar fitxers i directoris
Get-ChildItem

# Canviar de directori
Set-Location -Path "C:\Path\To\Directory"

# Crear un nou directori
New-Item -Path "C:\Path\To\NewDirectory" -ItemType Directory

Gestió de Processos

# Llistar processos en execució
Get-Process

# Aturar un procés pel nom
Stop-Process -Name "notepad"

Treballant amb Serveis

# Llistar tots els serveis
Get-Service

# Iniciar un servei
Start-Service -Name "wuauserv"

# Aturar un servei
Stop-Service -Name "wuauserv"

Automatització de Tasques amb Scripts de PowerShell

Exemple: Automatització d'un Procés de Còpia de Seguretat

# Definir els camins d'origen i destinació
$source = "C:\Path\To\Source"
$destination = "C:\Path\To\Backup"

# Crear una marca de temps
$timestamp = Get-Date -Format "yyyyMMddHHmmss"

# Crear un directori de còpia de seguretat amb la marca de temps
$backupDir = "$destination\Backup_$timestamp"
New-Item -Path $backupDir -ItemType Directory

# Copiar fitxers al directori de còpia de seguretat
Copy-Item -Path "$source\*" -Destination $backupDir -Recurse

# Mostrar el resultat
Write-Output "Còpia de seguretat completada amb èxit a $backupDir"

Integració de PowerShell amb Pipelines de CI/CD

PowerShell es pot integrar en pipelines de CI/CD per automatitzar processos de construcció, prova i desplegament.

Exemple: Utilitzant PowerShell en un Pipeline d'Azure DevOps

# azure-pipelines.yml
trigger:
- main

pool:
  vmImage: 'ubuntu-latest'

steps:
- task: UseDotNet@2
  inputs:
    packageType: 'sdk'
    version: '5.x'
    installationPath: $(Agent.ToolsDirectory)/dotnet

- script: |
    pwsh -File build.ps1
  displayName: 'Executar Script de PowerShell'

Exemple: Script build.ps1

# build.ps1
Write-Output "Iniciant el procés de construcció..."

# Exemple de passos de construcció
dotnet build

Write-Output "Procés de construcció completat."
