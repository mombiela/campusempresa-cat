Construir el Projecte

En aquesta secció, cobrirem com construir un projecte utilitzant PowerShell. Això inclourà configurar l'estructura del projecte, escriure scripts i organitzar el teu codi per a una millor llegibilitat i mantenibilitat. Al final d'aquesta secció, hauràs de ser capaç de crear un projecte de PowerShell des de zero i entendre les millors pràctiques per estructurar el teu codi.

Configurar l'Estructura del Projecte

Abans de començar a escriure qualsevol script de PowerShell, és important configurar una estructura de projecte adequada. Això ajuda a organitzar els teus scripts i facilita la gestió de projectes més grans.

Conceptes Clau

  • Directori del Projecte: La carpeta arrel on residiran tots els fitxers del teu projecte.
  • Carpeta d'Scripts: Una subcarpeta per emmagatzemar tots els teus scripts de PowerShell.
  • Carpeta de Mòduls: Una subcarpeta per emmagatzemar mòduls de PowerShell reutilitzables.
  • Fitxers de Configuració: Fitxers que emmagatzemen configuracions per als teus scripts.

Exemple d'Estructura del Projecte

Aquí tens un exemple d'una estructura típica de projecte:

MyPowerShellProject/
├── Scripts/
│   ├── script1.ps1
│   ├── script2.ps1
├── Modules/
│   ├── MyModule/
│   │   ├── MyModule.psm1
│   │   ├── MyModule.psd1
├── Config/
│   ├── settings.json
├── README.md

Escriure el Teu Primer Script

Ara que tenim la nostra estructura de projecte configurada, escrivim el nostre primer script de PowerShell.

Exemple d'Script

Crea un nou fitxer anomenat script1.ps1 dins de la carpeta Scripts i afegeix el següent codi:

# script1.ps1
Write-Host "Hola, Món!"

Explicació

  • Write-Host: Aquest cmdlet s'utilitza per mostrar missatges a la consola. En aquest cas, imprimirà "Hola, Món!" a la pantalla.

Organitzar el Teu Codi

A mesura que el teu projecte creix, és important mantenir el teu codi organitzat. Això es pot aconseguir desglossant els teus scripts en mòduls més petits i reutilitzables.

Crear un Mòdul

Els mòduls són una manera d'empaquetar codi de PowerShell reutilitzable. Creem un mòdul senzill.

  1. Crea una nova carpeta anomenada MyModule dins de la carpeta Modules.
  2. Dins de MyModule, crea dos fitxers: MyModule.psm1 i MyModule.psd1.

Exemple de Codi del Mòdul

Afegeix el següent codi a MyModule.psm1:

# MyModule.psm1
function Get-Greeting {
    param (
        [string]$Name
    )
    return "Hola, $Name!"
}

Manifest del Mòdul

Afegeix el següent codi a MyModule.psd1:

@{
    ModuleVersion = '1.0.0'
    GUID = 'd9b1a5e1-5c3b-4a6d-9b1a-5e15c3b4a6d9'
    Author = 'El Teu Nom'
    Description = 'Un mòdul de salutació senzill'
}

Utilitzar el Mòdul

Per utilitzar el mòdul en el teu script, necessites importar-lo. Afegeix el següent codi a script1.ps1:

# script1.ps1
Import-Module ../Modules/MyModule/MyModule.psm1
$greeting = Get-Greeting -Name "Alice"
Write-Host $greeting

Explicació

  • Import-Module: Aquest cmdlet s'utilitza per importar un mòdul al teu script.
  • Get-Greeting: Aquesta és la funció definida a MyModule.psm1. Pren un paràmetre Name i retorna un missatge de salutació.

Gestió de la Configuració

Gestionar les configuracions és crucial per a qualsevol projecte. PowerShell et permet emmagatzemar configuracions en diversos formats com JSON, XML o text pla.

Exemple de Fitxer de Configuració

Crea un nou fitxer anomenat settings.json dins de la carpeta Config i afegeix el següent contingut:

{
    "GreetingName": "Alice"
}

Llegir la Configuració en PowerShell

Afegeix el següent codi a script1.ps1 per llegir el fitxer de configuració:

# script1.ps1
Import-Module ../Modules/MyModule/MyModule.psm1

$configPath = "../Config/settings.json"
$config = Get-Content $configPath | ConvertFrom-Json

$greeting = Get-Greeting -Name $config.GreetingName
Write-Host $greeting

Explicació

  • Get-Content: Aquest cmdlet llegeix el contingut d'un fitxer.
  • ConvertFrom-Json: Aquest cmdlet converteix una cadena JSON en un objecte de PowerShell.

Conclusió

En aquesta secció, hem cobert els conceptes bàsics de configurar un projecte de PowerShell, escriure scripts, organitzar el codi en mòduls i gestionar les configuracions. Seguint aquestes millors pràctiques, pots crear projectes de PowerShell ben estructurats i mantenibles. Continua practicant afegint més scripts i mòduls al teu projecte, i explora temes avançats a mesura que et sentis més còmode amb PowerShell.
