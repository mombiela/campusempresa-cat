Introducció a la Infraestructura com a Codi (IaC)

  • Definició: La Infraestructura com a Codi (IaC) és el procés de gestionar i aprovisionar la infraestructura informàtica a través de scripts llegibles per màquines, en lloc de mitjançant la configuració de maquinari físic o eines de configuració interactives.
  • Beneficis:
    • Consistència i repetibilitat
    • Control de versions
    • Automatització i eficiència
    • Reducció del risc d'errors humans

Començant amb PowerShell

  • Què és PowerShell?: Un marc d'automatització de tasques i gestió de configuració de Microsoft, que consisteix en una línia de comandes i un llenguatge de script associat.

  • Instal·lant PowerShell:

    • A Windows: PowerShell ve preinstal·lat.
    • A macOS/Linux: Utilitza gestors de paquets com Homebrew o apt-get.
    # macOS
brew install --cask powershell

# Linux (Ubuntu)
sudo apt-get install -y powershell

Sintaxi bàsica de PowerShell

  • Cmdlets: Comandes de PowerShell, pronunciades "command-lets".

    • Exemple: Get-Command, Get-Help, Get-Process
    # Llistar tots els cmdlets disponibles
Get-Command

# Obtenir ajuda per a un cmdlet específic
Get-Help Get-Process

  • Variables: Emmagatzemar dades que es poden utilitzar i manipular.

    $greeting = "Hola, Món!"
Write-Output $greeting

  • Pipelines: Passar la sortida d'un cmdlet com a entrada a un altre.

    Get-Process | Sort-Object CPU -Descending

Escriure scripts de PowerShell

  • Conceptes bàsics dels scripts: Els scripts de PowerShell són fitxers de text amb una extensió .ps1.

  • Executar scripts:

    • Assegura't que la política d'execució permeti l'execució d'scripts: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned
    • Executar un script: .\script.ps1
    # Exemple de script: hello.ps1
$name = "Usuari de PowerShell"
Write-Output "Hola, $name!"

Infraestructura com a Codi amb PowerShell

  • PowerShell Desired State Configuration (DSC):
    • Definició: Una plataforma de gestió en PowerShell que et permet gestionar la teva infraestructura IT i de desenvolupament amb configuració com a codi.
    • Components:
      • Configuració: Defineix l'estat desitjat.
      • Recursos: Els blocs de construcció de DSC, com ara fitxers, serveis i paquets.
      • Local Configuration Manager (LCM): El motor que aplica la configuració.

Escriure una configuració bàsica de DSC

  • Exemple: Assegurar que un fitxer existeixi amb un contingut específic.

    Configuration SampleConfig {
    Node "localhost" {
        File ExampleFile {
            DestinationPath = "C:\Example\example.txt"
            Contents = "Hola, DSC!"
            Ensure = "Present"
        }
    }
}

# Generar el fitxer MOF
SampleConfig

# Aplicar la configuració
Start-DscConfiguration -Path .\SampleConfig -Wait -Verbose

Configuracions avançades de DSC

  • Configuracions compostes: Configuracions reutilitzables que es poden incloure en altres configuracions.

  • Configuracions parcials: Permeten que diferents equips gestionin diferents parts de la configuració.

    Configuration WebServerConfig {
    Import-DscResource -ModuleName PSDesiredStateConfiguration

    Node "WebServer" {
        WindowsFeature IIS {
            Name = "Web-Server"
            Ensure = "Present"
        }

        File WebContent {
            DestinationPath = "C:\inetpub\wwwroot\index.html"
            Contents = "<html><body>Hola, Món!</body></html>"
            Ensure = "Present"
        }
    }
}

# Generar i aplicar la configuració
WebServerConfig
Start-DscConfiguration -Path .\WebServerConfig -Wait -Verbose

Millors pràctiques per a IaC amb PowerShell

  • Control de versions: Utilitza Git o un altre sistema de control de versions per fer un seguiment dels canvis.

  • Modularització: Divideix les configuracions en mòduls més petits i reutilitzables.

  • Proves: Utilitza Pester, un marc de proves per a PowerShell, per escriure proves per a les teves configuracions.

    Describe "SampleConfig" {
    It "Hauria de crear el fitxer d'exemple" {
        $filePath = "C:\Example\example.txt"
        Test-Path $filePath | Should -Be $true
    }
}

Conclusió

La Infraestructura com a Codi amb PowerShell, especialment utilitzant DSC, permet una gestió robusta, repetible i automatitzada de la teva infraestructura. Aprofitant les capacitats de script de PowerShell, pots assegurar-te que la teva infraestructura estigui sempre en l'estat desitjat, reduint el risc de desviació de configuració i errors humans. A mesura que avancis, considera integrar aquestes pràctiques amb pipelines de CI/CD per millorar encara més els teus fluxos de treball d'automatització.
