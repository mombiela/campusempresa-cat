Introducció

PowerShell és un llenguatge de script potent i una línia d'ordres dissenyada per a l'automatització de tasques i la gestió de configuracions. Per treure el màxim profit de PowerShell, és essencial seguir les millors pràctiques que assegurin que els teus scripts siguin eficients, mantenibles i segurs. Aquesta secció cobrirà les millors pràctiques per escriure scripts de PowerShell, des de nivells principiants fins a avançats.

Millors Pràctiques Generals

  • Utilitza Noms Descriptius: Sempre utilitza noms descriptius per a variables, funcions i scripts per fer que el teu codi sigui més llegible.
  • Comenta el Teu Codi: Utilitza comentaris per explicar el propòsit del teu codi, especialment la lògica complexa.
  • Format Consistent: Segueix un estil consistent per a la indentació, espaiat i claus per millorar la llegibilitat.
  • Gestió d'Errors: Implementa una gestió d'errors adequada per fer que els teus scripts siguin robustos i fàcils d'utilitzar.

Exemple

# Aquesta funció calcula el factorial d'un número
function Get-Factorial {
    param (
        [int]$Number
    )
    
    if ($Number -lt 0) {
        throw "El número ha de ser no negatiu"
    }

    $result = 1
    for ($i = 1; $i -le $Number; $i++) {
        $result *= $i
    }

    return $result
}

try {
    $factorial = Get-Factorial -Number 5
    Write-Output "Factorial: $factorial"
} catch {
    Write-Error "S'ha produït un error: $_"
}

Estructura de l'Script

  • Disseny Modular: Divideix els teus scripts en funcions més petites i reutilitzables.
  • Validació de Paràmetres: Utilitza la validació de paràmetres per assegurar-te que les teves funcions reben entrades vàlides.
  • Gestió de Sortides: Utilitza Write-Output, Write-Error, Write-Verbose i Write-Debug adequadament per gestionar diferents tipus de sortides.

Exemple

function Get-Square {
    param (
        [Parameter(Mandatory=$true)]
        [int]$Number
    )

    return $Number * $Number
}

# Script principal
try {
    $square = Get-Square -Number 4
    Write-Output "Quadrat: $square"
} catch {
    Write-Error "S'ha produït un error: $_"
}

Millors Pràctiques de Seguretat

  • Evita Codificar Credencials: Utilitza mètodes segurs per gestionar credencials, com ara Get-Credential.
  • Principi de Menor Privilegi: Executa els teus scripts amb els privilegis mínims necessaris.
  • Signatura de Codi: Signa els teus scripts per assegurar la seva integritat i autenticitat.

Exemple

# Sol·licita credencials de manera segura
$credentials = Get-Credential

# Utilitza les credencials de manera segura
Invoke-Command -ComputerName "Server01" -Credential $credentials -ScriptBlock {
    # El teu codi aquí
}

Optimització del Rendiment

  • Utilitza Cmdlets en Lloc de Comandes Externes: Els cmdlets estan optimitzats per al rendiment i s'han de preferir sobre les comandes externes.
  • Evita Bucles Innecessaris: Utilitza operacions de pipeline i cmdlets integrats per gestionar col·leccions de manera eficient.
  • Mesura el Rendiment: Utilitza Measure-Command per mesurar el rendiment dels teus scripts.

Exemple

# Mesura el temps que triga a executar-se una comanda
$time = Measure-Command {
    Get-Process
}
Write-Output "Temps emprat: $($time.TotalSeconds) segons"

Tècniques Avançades

  • Splatting: Utilitza splatting per passar paràmetres a funcions i cmdlets de manera neta i llegible.
  • Mòduls Personalitzats: Crea mòduls personalitzats per organitzar i distribuir les teves funcions.
  • Funcions Avançades: Utilitza funcions avançades per aprofitar característiques com el binding de cmdlets i paràmetres comuns.

Exemple

# Exemple de splatting
$params = @{
    Name = "Notepad"
    ComputerName = "localhost"
}
Get-Process @params

# Exemple de funció avançada
function Get-AdvancedFactorial {
    [CmdletBinding()]
    param (
        [Parameter(Mandatory=$true)]
        [int]$Number
    )

    if ($Number -lt 0) {
        throw "El número ha de ser no negatiu"
    }

    $result = 1
    for ($i = 1; $i -le $Number; $i++) {
        $result *= $i
    }

    return $result
}

Conclusió

Seguir les millors pràctiques en la scriptació de PowerShell assegura que el teu codi sigui eficient, mantenible i segur. Utilitzant noms descriptius, comentant el teu codi, gestionant errors adequadament i adherint-te a les pautes de seguretat i rendiment, pots crear scripts robustos que siguin fàcils d'entendre i mantenir. A mesura que avancis, incorporar tècniques com splatting, mòduls personalitzats i funcions avançades millorarà encara més les teves capacitats de scriptació.
